## 1. Контекст задачи

**Входной объём** — комплексный тензор после гетеродина размером **16 × 16 × 10 000**:

| Ось | Размер | Физический смысл |
|-----|--------|-----------------|
| X   | 16     | Приёмная апертура (горизонталь) |
| Y   | 16     | Приёмная апертура (вертикаль) |
| Z   | 10 000 | Дальность (отсчёты по времени) |

**Цель пайплайна:**
- За минимальное время найти кандидатов в цели (grobe дальность + направление)
- Послать пучок FM-m сигналов в область кандидата
- На подтверждённых кандидатах выполнить точный анализ

**Данные:** сигнал после гетеродина (I/Q, комплексная огибающая в базовой полосе), **не** сырые RF-данные.

## 2. Скользящий FFT по дальности с перекрытием (шаг 8)

### 2.1 Математика

Для длины окна **W = 16** и шага **s = 8** (overlap 50%) количество матриц по дальности:

$$N_{frames} = \left\lfloor \frac{N_Z - W}{s} \right\rfloor + 1 = \left\lfloor \frac{10000 - 16}{8} \right\rfloor + 1 = \mathbf{1250}$$

Для каждого окна *n* (позиция $z_n = n \cdot s$) выполняется 3D FFT:

$$F_n(k_x, k_y, k_z) = \text{FFT}_{3D}\bigl[f(x, y, z_n : z_n + W)\bigr], \quad n = 0 \ldots 1249$$

### 2.2 Почему overlap 50%?

| Параметр | Без перекрытия (шаг 16) | С перекрытием (шаг 8) |
|----------|------------------------|----------------------|
| Кол-во матриц | 625 | **1250** |
| Ошибка локализации | ±8 отсчётов | **±4 отсчёта** |
| Вычислительная нагрузка | ×1 | ×2 |
| Вероятность обнаружения | базовая | **выше** |

При 50% overlap каждый отсчёт по дальности участвует в двух соседних окнах — разрешение по дальности вдвое лучше при умеренном росте вычислений.

### 2.3 Карта энергии

После 3D FFT для каждого куба извлекается **карта энергии 16×16**:

$$E_n(k_x, k_y) = \max_{k_z}\bigl|F_n(k_x, k_y, k_z)\bigr|^2$$

Это вход для AI-детектора — компактный признак размером **16×16** вместо куба 16×16×16.

### 2.4 GPU-реализация (rocFFT batch mode)

```cpp
// Все 1250 кубов за один вызов rocFFT
size_t lengths[3] = {16, 16, 16};
size_t batch = 1250;

rocfft_plan_create(&plan,
    rocfft_placement_inplace,
    rocfft_transform_type_complex_forward,
    3, lengths, batch, info);

rocfft_execute(plan, &in_buf, &out_buf, info);

// Кастомный HIP кернел: max(|F|²) по k_z → 1250 матриц 16×16
```

## 3. Извлечение азимута, угла места и дальности

### 3.1 Из матрицы 16×16 после FFT

Пик в пространственно-частотной плоскости $E_n(k_x, k_y)$ даёт углы прихода (direction cosines):

$$\sin\theta_{az} = \frac{k_x^{peak}}{N_x} \cdot \frac{\lambda}{d_x}, \qquad \sin\theta_{el} = \frac{k_y^{peak}}{N_y} \cdot \frac{\lambda}{d_y}$$

где $d_x, d_y$ — шаг между элементами апертуры, $\lambda$ — длина волны.

При $N_x = N_y = 16$ угловое разрешение — **1/16 апертуры** — достаточно для грубого наведения FM-m луча.

### 3.2 Дальность

$$R_{coarse} = \left(n^* \cdot s + \arg\max_{k_z}\bigl|F_{n^*}\bigr|\right) \cdot \Delta r$$

где $n^*$ — индекс окна с максимальной энергией, $\Delta r$ — метровый эквивалент одного отсчёта.

### 3.3 Точность первого этапа

| Величина | Погрешность (шаг 8) |
|----------|-------------------|
| Дальность | ±4 отсчёта = $\pm 4 \cdot \Delta r$ |
| Азимут | ±1 бин из 16 ≈ ±6.25% апертуры |
| Угол места | ±1 бин из 16 ≈ ±6.25% апертуры |

## 4. Стратегия посылки пучка FM-m (8–10 сигналов)

Вместо одного точного луча посылается **пучок из 8–10 FM-m сигналов**, перекрывающих конус неопределённости вокруг $(\theta_{az}, \theta_{el})$.

| Параметр | Значение |
|----------|---------|
| Угловая погрешность coarse | ±1 бин из 16 ≈ ±6.25% апертуры |
| Дальностная погрешность | ±4 отсчёта при overlap 50% |
| Кол-во лучей в пучке | 8–10 |
| Шаг между лучами в пучке | ≈ 1 бин FFT по углу |
| Цель | Покрыть область неопределённости с запасом |

**Логика:** пучок → отклик → уточнение → второй этап (точный анализ только в подтверждённой области). Это когнитивная стратегия: grobe scan → targeted probe → fine analysis.

## 5. AI-детектор: где применять и какие варианты

### 5.1 Место AI в пайплайне

Оптимальная точка — **после суммирования 3D FFT**: вход — матрицы $16 \times 16$ или последовательность 1250 таких матриц. Сеть работает на готовом компактном признаке (пространственный спектр), а не на сырых данных.

### 5.2 Вариант A: 2D CNN на матрицах 16×16 ⭐ Рекомендуемый

**Вход:** матрица $|F_n|^2$ размером $16 \times 16$, нормированная, 1 или 2 канала (мощность / Re+Im).

**Архитектура (< 200 KB весов):**

```
Input:  16×16×1
Conv2D(4 filters, 5×5) → BN → ReLU      # 100 ops/pixel
MaxPool(2×2) → 8×8×4
Conv2D(8 filters, 3×3) → BN → ReLU      # 88 ops/pixel
GlobalAvgPool → Dense(32) → Dense(2)    # target / no-target
```

**Скорость:** инференс 1250 матриц батчем — **< 0.2 мс** на GPU.

### 5.3 Вариант B: 1D детектор аномалий по дальности

**Вход:** скалярный ряд длиной 1250 — суммарная энергия каждого куба $e_n = \sum_{k_x,k_y} E_n$.

| Модель | Размер | Латентность | Плюсы |
|--------|--------|------------|-------|
| 1D-CNN (3 слоя) | ~50 KB | ~10 мкс | Простота |
| LSTM Autoencoder | ~200 KB | ~50 мкс | Учитывает контекст |
| Sliding threshold | 0 KB | ~1 мкс | Базовый бенчмарк |

**Минус:** теряется информация об угле, только дальность.

### 5.4 Вариант C: 3D CNN на пространственно-угловой последовательности

**Вход:** тензор $16 \times 16 \times 16$ (небольшое окно соседних матриц).

```
Input:  16×16×16
Conv3D(8 filters, 3×3×3) → ReLU
MaxPool3D(2×2×2) → 8×8×8×8
Conv3D(16 filters, 3×3×3) → ReLU
GlobalAvgPool → Dense(2)
```

**Плюс:** ловит структуру сигнала по всем трём осям одновременно.  
**Минус:** нужно больше данных для обучения.

### 5.5 Вариант D: Гибрид (рекомендуемая итоговая архитектура)

```
[Energy maps 16×16 × 1250 матриц]
         ↓                ↓
 [2D CNN per matrix]   [1D energy trace]
 [target/no-target]   [range anomaly score]
         ↓                ↓
     [Fusion: AND / OR]
         ↓
 [Candidates: (n*, kx*, ky*) → R, az, el]
```

Двухпоточный подход: один детектор отвечает за "**что**" (пространственная структура), другой за "**где по дальности**" (аномалия энергии).

## 6. GPU-реализация ROCm/HIP (RX 9070)

### 6.1 Fused Kernel: FFT → Energy → Inference

Ключевое ускорение — объединить все три шага без выгрузки в глобальную память:

```cpp
// HIP kernel: один workgroup обрабатывает один куб 16×16×16
__global__ void fused_detect(
    complex_float* in,
    float*         energy_out,
    float*         weights_conv,
    float*         detect_out)
{
    __shared__ complex_float lds[16*16*16];  // 32 KB LDS

    // 1. Загрузка куба в LDS
    // 2. Inline 3D FFT (16-point radix-2 по каждой оси)
    // 3. |F|² → energy map 16×16 (sum over k_z)
    // 4. Применение весов CNN (16×16 → 4 нейрона → 1 выход)
    // Результат: один float в detect_out[blockIdx]
}

// Запуск: blockDim=256, gridDim=1250 → все кубы за один dispatch
```

### 6.2 Оценка времени на RX 9070 (16 GB VRAM)

| Этап | Операции | Время (оценка) |
|------|----------|---------------|
| rocFFT batch 1250 × 16³ | ~800M flops | ~0.5 мс |
| Energy map reduction | ~50M flops | ~0.1 мс |
| CNN inference 1250 × 16×16 | ~200M flops | ~0.2 мс |
| 1D аномалия по 1250 точкам | ~5M flops | < 0.05 мс |
| **Итого coarse detection** | | **~1 мс** |
| Глубокий анализ без GPU | | > 100 мс |

**Ускорение:** 100–500× по объёму вычислений относительно полного точного анализа.

### 6.3 Размер данных в VRAM

```
Входной буфер:  16 × 16 × 10000 × 8 байт (complex float) =  20 MB
FFT буфер:      1250 × 16³ × 8                            =  51 MB
Energy maps:    1250 × 16 × 16 × 4                        =   5 MB
Веса CNN:                                                 ~0.2 MB
─────────────────────────────────────────────────────────────────
Итого:                                                   ~76 MB
Доступно VRAM:                                        16 000 MB ✓
```

## 7. Стратегия обучения AI-детектора

### 7.1 Синтетический датасет

Генерируется непосредственно на GPU (кастомный кернел):

| Тип примера | Описание | Кол-во |
|------------|----------|--------|
| Положительные | Куб с FM-m сигналом, SNR 3–25 dB | 50 000–100 000 |
| Негативные (шум) | Тепловой шум | 30 000–60 000 |
| Негативные (помеха) | Заградительные + cluttered реплики | 20 000–40 000 |
| Аугментация | Случайные фазы, Доплер, неполная апертура | ×3–5 |

**Время генерации:** несколько минут на RX 9070.

### 7.2 Задачи и метки

| Задача | Метки | Выход сети |
|--------|-------|-----------|
| Бинарная детекция | 0/1 (цель / нет) | Dense(1) + sigmoid |
| Оценка SNR | float | Dense(1) + relu |
| Грубый угол | $(k_x^{peak}, k_y^{peak})$ | Dense(2) |
| Мультизадача | все выше | Multi-head output |

Мультизадачное обучение позволяет сети одновременно подтвердить наличие цели **и** уточнить направление в одном прямом проходе.

## 8. Полная схема двухэтапного пайплайна

```
┌──────────────────────────────────────────────────────────────┐
│  ЭТАП 1: COARSE DETECTION  (~1 мс на GPU)                    │
│                                                              │
│  16×16×10000  ──►  Нарезка 1250 кубов (overlap 50%, шаг 8)  │
│                              │                              │
│                    rocFFT batch 3D (1250×16³)               │
│                              │                              │
│               Energy maps 16×16 × 1250  (5 MB)             │
│                    ┌─────────┴──────────┐                   │
│              2D CNN per matrix      1D energy trace         │
│              (target / no-target)   (anomaly score)         │
│                    └─────────┬──────────┘                   │
│                    Fusion & threshold                        │
│                              │                              │
│              Кандидаты: [(n*, kx*, ky*), ...]               │
└──────────────────────────────────────────────────────────────┘
                              │
┌──────────────────────────────────────────────────────────────┐
│  ЗОНДИРОВАНИЕ FM-m  (активная фаза)                          │
│                                                              │
│  Для каждого кандидата:                                      │
│    (kx*, ky*)  →  (az, el)  →  пучок 8–10 FM-m лучей        │
│     n* × 8    →  R_coarse  →  диапазон дальности            │
└──────────────────────────────────────────────────────────────┘
                              │
┌──────────────────────────────────────────────────────────────┐
│  ЭТАП 2: FINE ANALYSIS  (только на кандидатах)               │
│                                                              │
│  Исходный фрагмент ±N отсчётов вокруг R_coarse              │
│  → Полный 3D FFT + CFAR + MUSIC/ESPRIT                      │
│  → Точные (R, az, el, v_doppler)                            │
└──────────────────────────────────────────────────────────────┘
```

## 9. Выводы и рекомендации

### Скользящий FFT (шаг 8, overlap 50%)
Оптимальный выбор: вдвое лучше локализует всплеск по сравнению с шагом 16, количество матриц остаётся в пределах **1250** — ровно в рамках ограничения.

### AI на матрицах 16×16
Правильная точка применения — **после 3D FFT**: сеть получает компактный, информативный признак.  
**Рекомендованный вариант — гибрид D:**
- 2D CNN по матрицам — пространственная структура (что и откуда)
- 1D аномальный детектор — энергетический след (где по дальности)
- Объединение через AND/OR

### Посылка пучка FM-m (8–10 лучей)
Покрытие конуса неопределённости ±1–2 угловых бина даёт высокую вероятность попадания — стандартная практика когнитивного радара (Cognitive Radar / Perception-Action Cycle).

### RX 9070 (ROCm)
Весь первый этап укладывается в **~1 мс** при батчевой обработке через rocFFT + fused HIP-кернел. Данные не покидают VRAM между этапами. Итоговое ускорение относительно CPU-расчёта — **100–500×**.
