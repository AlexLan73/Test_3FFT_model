# Полный обзор радарного пайплайна
## 3D FFT, 1250 окон по дальности и гибридный каскадный классификатор

**Автор:** Александр Ланин  
**Дата:** 07.07.2026  
**Платформа:** AMD RX 9070 (16 GB VRAM) · ROCm · HIP · rocFFT

---

## 1. Геометрия данных и 3D FFT

Исходные данные после гетеродина — комплексный тензор:

- Размер: **16×16×10000**
- Оси: X (16), Y (16) — апертура; Z (10000) — дальность (отсчёты по времени)

По оси Z берём скользящее окно длиной **W = 16** с шагом **s = 8** (overlap 50%).  
Число окон по дальности:

\[ N_{frames} = \left\lfloor \frac{N_Z - W}{s} \right\rfloor + 1 = \left\lfloor \frac{10000 - 16}{8} \right\rfloor + 1 = 1250 \] [1]

Каждое окно даёт куб **16×16×16**, на котором выполняется 3D FFT.

```text
16×16×10000
   → Нарезка с overlap 50%, шаг 8
   → 1250 кубов 16×16×16
   → 3D FFT (rocFFT) → спектр 16×16×16
   → Свёртка по kz (max/sum) → 1250 матриц 16×16
```

Свёртка по kz:

\[ E_n(k_x,k_y) = \max_{k_z} \bigl|F_n(k_x, k_y, k_z)\bigr|^2 \quad \text{или} \quad E_n(k_x,k_y) = \sum_{k_z} \bigl|F_n(k_x, k_y, k_z)\bigr|^2 \]

Итого: **1250 окон по дальности** → **1250 плоских матриц 16×16**, каждая отражает
распределение энергии по углам (kx, ky) для своего окна.

## 2. Coarse detection: быстрый фильтр по 1250 окнам

На первом уровне делаем минимально дорогой анализ:

1. Для каждой матрицы 16×16 считаем суммарную энергию:

   \[ E_n = \sum_{k_x,k_y} E_n(k_x,k_y) \]

2. Строим адаптивный порог CFAR по всем 1250 окнам (\(\mu_{noise}, \sigma_{noise}\)).
3. Окна с \(E_n > \mu_{noise} + \kappa \cdot \sigma_{noise}\) считаются кандидатами.

Результат:

- ~90–95% окон сразу классифицируются как шум и отбрасываются;
- остаётся **50–100 окон-кандидатов** для более глубокого анализа.

Вычислительно — это только сумма и сравнение порога → < 0.01 мс на батч 1250.

## 3. Уровень 2: объёмный классификатор 16×16×32

Для каждого окна-кандидата n:

1. Берём расширенный куб 16×16×32 из сырых данных с перекрытием:

   ```text
   z_start = n* × 8 - 8
   z_end   = z_start + 32
   cube    = volume[:, :, z_start:z_end]  # 16×16×32
   ```

2. Умножаем на пространственные и временные окна (фильтры):

   \[ g(x,y,z) = f(x,y,z) \cdot w(x) \cdot w(y) \cdot h(z) \]

   - w(x), w(y) — окна по апертуре (Чебышёв, Хэмминг);
   - h(z) — окно по дальности (Хэнн, Блэкман).

3. Выполняем 3D FFT куба 16×16×32 и считаем \(|F|^2\).
4. Подаём тензор \(|F|^2\) на 3D-CNN-классификатор.

Задачи Уровня 2:

- подтвердить/отвергнуть кандидата;
- оценить \((k_x^*, k_y^*)\) — азимут и угол места;
- оценить \(k_z^*\) — положение внутри патча 32;
- отделить цель от очевидной помехи.

---

## 4. Уровень 3: глубокий анализ 16×16×256

Для небольшого числа сложных кандидатов (~5 из 1250):

1. Берём длинный фрагмент 16×16×256 из сырых данных вокруг грубой дальности;
2. Применяем классические методы (MUSIC, ESPRIT) для точной оценки углов и дальности;
3. При необходимости используем Conv3D-классификатор для разделения "цель / заградитель / шум";
4. Анализируем форму огибающей для идентификации типа сигнала.

Этот уровень тяжёлый по вычислениям (5–10 мс на фрагмент), но запускается
редко, поэтому не доминирует в общей задержке.

## 5. Гибридный классификатор на Уровне 2

На уровне анализа 16×16 (или 16×16×32) разумно использовать **гибрид из
трёх ветвей**:

1. **Matched Filter (MF)** — классический согласованный фильтр по углам,
   не требует обучения, хорош при высоком SNR;
2. **Tiny MLP / 2D-CNN** — компактная обучаемая сеть (< 150 KB весов),
   устойчивее к шуму и сложным помехам;
3. **Autoencoder (AE)** — автоэнкодер, обученный только на шуме, ловит
   любые аномалии без меток.

Объединение:

```text
MF_score ∈ [0,1]   # классический отклик согласованного фильтра
MLP_prob ∈ [0,1]   # вероятность цели от MLP/CNN
MSE      ∈ [0,∞)   # ошибка реконструкции автоэнкодера

final_score = w1·MF_score + w2·MLP_prob + w3·f(MSE)
```

Веса \(w_1, w_2, w_3\) зависят от оценённого SNR:

- при SNR > 15 dB доминирует MF (w1≈0.7);
- при 7–15 dB доминирует MLP (w2≈0.6);
- при SNR < 7 dB увеличиваем вес AE (w3≈0.4).

Решение по окну:

```text
final_score > threshold_high  → уверенная цель → в список для FM-m пучка
final_score > threshold_low   → слабый кандидат → накопить соседние окна
final_score ≤ threshold_low   → шум → окно отбрасывается
```

## 6. Точки связи с FM-m пучком

После Уровня 2 (или Уровня 3 для сложных случаев) для каждого подтверждённого
кандидата имеем:

- грубый номер окна по дальности n*;
- уточнённую позицию внутри патча \(k_z^*\);
- углы \((k_x^*, k_y^*)\) → азимут и угол места.

Дальше формируем пучок из 8–10 FM-m лучей, покрывающий конус неопределённости
по углам (±1–2 угловых бина) и небольшой диапазон по дальности вокруг \(R_{coarse}\).

Это позволяет:

- избежать полного сканирования апертуры и дальности;
- сфокусировать энергию на небольшой области пространства;
- использовать когнитивную стратегию: "обнаружил → зондировал → уточнил".

## 7. Производительность и память

Оценка для RX 9070 (16 GB VRAM):

- 3D FFT по 1250 кубам 16×16×16: ~0.5 мс;
- свёртка по kz и энергетический фильтр: < 0.1 мс;
- анализ 50–100 кандидатов на Уровне 2 (Conv3D + гибрид MF/MLP/AE): ~0.2–0.3 мс;
- fusion и пороговая логика: < 0.01 мс;
- Уровень 3: редко, 5–10 мс на 1 фрагмент (×5 фрагментов ≈ 25–50 мс).

**Coarse detection (Уровень 1+2)** укладывается примерно в **~0.8–1.0 мс**.  
Глубокий анализ Уровня 3 не запускается для каждого окна, поэтому не доминирует
в общей задержке.

По памяти весь пайплайн (данные + веса сети) занимает менее 100 MB из 16 GB VRAM.

## 8. Сводные выводы

1. **1250 окон по дальности** — удобная дискретизация для coarse detection,
   каждая матрица 16×16 несёт информацию о пространственном распределении энергии
   в своём окне.

2. **Каскад "быстро → средне → глубоко"** резко снижает вычислительную
   нагрузку: 95% окон отбрасываются на первом этапе, тяжёлый анализ выполняется
   только для единиц кандидатов.

3. **Пространство паттернов конечное и структурированное**, поэтому хватит
   синтетического датасета 50–100 тысяч примеров для обучения MLP/CNN и AE.

4. **Гибрид Matched Filter + MLP + Autoencoder** даёт устойчивую детекцию по
   всему диапазону SNR и типов помех, объединяя интерпретируемость классики
   и адаптивность нейросетей.

5. **Overlap 50% по оси Z** критичен: гарантирует, что цель не потеряется на
   границе окон и попадёт целиком хотя бы в один патч 16×16×16 или 16×16×32.
