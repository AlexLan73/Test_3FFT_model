## 1. Исправленный поток данных

```
16×16×10000  (комплексный тензор после гетеродина)
     │
     ▼  нарезка с overlap 50%, шаг 8
1250 кубов 16×16×16
     │
     ▼  rocFFT batch 3D → спектр 16×16×16 (комплексный)
     │
     ▼  свёртка по оси kz: max|F(kx,ky,kz)|² по всем kz
     │
1250 плоских матриц 16×16  ← вход классификатора
```

**Ключевой момент:** после 3D FFT мы сворачиваем (проецируем) куб спектра по оси
дальности kz. Получаем **плоскую матрицу энергии 16×16**, где каждый элемент
E_n(kx, ky) отвечает на вопрос: *с какого направления пришёл самый сильный сигнал
в данном дальностном окне n?*

---

## 2. Варианты свёртки по kz

| Операция | Формула | Физический смысл |
|----------|---------|-----------------|
| Max      | `max_{kz} |F|²`    | Самый яркий бин по дальности |
| Sum      | `sum_{kz} |F|²`    | Суммарная энергия (некогерентное накопление) |
| Argmax   | `argmax_{kz} |F|²` | Грубая дальность внутри куба (хранить отдельно) |

**Рекомендация:** хранить оба значения:

```
E_n(kx, ky)    = max_{kz} |F_n(kx,ky,kz)|²  → плоская матрица для детектора
kz_n*(kx, ky)  = argmax_{kz} |F_n|²           → уточнение дальности
```

Итоговая точная дальность до цели:

```
R = (n* × 8 + kz*) × Δr
```

где n* — номер куба с максимальной энергией, kz* — бин внутри куба.

## 3. Пространство паттернов матрицы 16×16

После свёртки по kz входное пространство классификатора:

- **Размерность входа:** 16×16 = 256 вещественных чисел (энергия)
- **Число различимых направлений:** 16×16 = 256 угловых бинов (kx, ky)
- **Число кубов по дальности:** 1250

Точечная цель в матрице 16×16 даёт **один доминирующий пик** в бине (kx*, ky*).

### Реальное число уникальных паттернов

| Фактор | Вариантов | Примечание |
|--------|-----------|-----------|
| Позиция пика (kx, ky) | 256 | Угловые бины 16×16 |
| Уровень SNR | 8–10 уровней | от -5 до 25 dB |
| Тип шума | 3 варианта | тепловой, заградительный, кластерный |
| Уровень помехи | 5 уровней | 0–30 dB выше шума |
| **Итого уникальных** | **~38 400** | Полное покрытие пространства |

**Вывод:** это не задача ImageNet с миллионами случайных изображений.
Пространство **закрытое и физически детерминированное** — огромный датасет не нужен.
Реальный датасет: **50 000–100 000 примеров** — генерируется за минуты на GPU.

## 4. Гибридная архитектура классификатора

### Обзор трёх ветвей

```
Входная матрица 16×16 (energy map)
  ┌─────────────────┬──────────────────┬───────────────────┐
  │  ВЕТКА 1        │  ВЕТКА 2         │  ВЕТКА 3          │
  │  Matched Filter │  Tiny MLP        │  Autoencoder      │
  │  (без обучения) │  (обучаемая)     │  (аномалии)       │
  │  score ∈ [0,1]  │  p ∈ [0,1]       │  mse ∈ [0,∞)      │
  └────────┬────────┴────────┬─────────┴────────┬──────────┘
           └─────────────────┼──────────────────┘
                             ▼
              final_score = w1·MF + w2·MLP + w3·f(MSE)
                             ▼
              score > threshold → ЦЕЛЬ ОБНАРУЖЕНА
              позиция = (kx*, ky*) из MLP + kz* + n*
```

---

## 5. Ветка 1: Matched Filter (классический)

### Принцип

Идеальная матрица 16×16 для цели в бине (kx*, ky*) — это **дельта-функция**:
единица в позиции (kx*, ky*), нули везде. Детекция = нормированная корреляция
входной матрицы со всеми 256 эталонами.

На GPU это матричное умножение:

```
query = flatten(E_n)         # вектор 256
D     = eye(256)             # матрица эталонов 256×256 (тождественная)
scores = D @ query           # 256 скоров → argmax → (kx*, ky*)
```

В реальности D не единичная — эталоны учитывают диаграмму направленности
антенны (PSF — Point Spread Function), поэтому пик размазан по нескольким бинам.

### Когда работает хорошо

- SNR > 10 dB
- Одна цель в кубе (нет перекрытия лепестков)
- Известная форма PSF

### Ограничения

- Деградирует при SNR < 10 dB (пик тонет в шуме)
- Не справляется с несколькими близкими целями
- Не адаптируется к неизвестным помехам

## 6. Ветка 2: Tiny MLP (обучаемый классификатор)

### Архитектура

```
Input: 16×16 = 256 (нормированная energy map)
  │
  Dense(128, ReLU) + BatchNorm
  │
  Dense(64, ReLU) + Dropout(0.2)
  │
  ├── Head 1 (detection):  Dense(1, sigmoid)    # есть цель / нет
  ├── Head 2 (angle):      Dense(2, linear)     # (kx*, ky*) в [0,15]
  └── Head 3 (range bin):  Dense(1, linear)     # kz* внутри куба
```

**Размер весов:** ~150 KB — помещается в L2-кеш GPU целиком.  
**Инференс батча 1250:** < 0.1 мс на RX 9070.

### Мультизадачное обучение

```python
loss = (
    1.0 * BCE(pred_detect, label_detect)     # детекция
  + 0.5 * MSE(pred_angle, label_angle)       # угол (только если цель есть)
  + 0.5 * MSE(pred_range, label_range)       # дальность внутри куба
)
```

### Генерация датасета (на GPU)

```python
# Синтетический датасет: всё на GPU, за минуты
for kx in range(16):
    for ky in range(16):
        for snr_db in [-5, 0, 5, 10, 15, 20, 25]:
            # Позитивный пример: цель в (kx, ky)
            cube = noise_cube() + target_cube(kx, ky, snr_db)
            F = fft3d(cube)
            E = max_over_kz(abs(F)**2)
            # Нормировка и добавление в датасет

# Негативный пример: только шум
cube = noise_cube()
E = max_over_kz(abs(fft3d(cube))**2)
```

### Когда работает лучше Matched Filter

- SNR 3–10 dB (низкий сигнал)
- Несколько слабых целей
- Нестационарный шум / переменная помеха
- Неточно известная PSF антенны

## 7. Ветка 3: Autoencoder (аномальный детектор)

### Принцип

Обучается **только на чистом шуме** — без меток, без знания о целях.
При появлении любого "нешумового" паттерна ошибка реконструкции MSE резко растёт.

```
Input 256 → Encoder → latent(16) → Decoder → Output 256
Loss = MSE(input, output)   # только на шуме!

Инференс:
  MSE < θ   →  чистый шум (цели нет)
  MSE > θ   →  АНОМАЛИЯ (цель, неизвестная помеха, сбой)
```

### Архитектура

```
Encoder:
  Dense(256→128, ReLU)
  Dense(128→64,  ReLU)
  Dense(64→16,   ReLU)   ← latent space

Decoder:
  Dense(16→64,  ReLU)
  Dense(64→128, ReLU)
  Dense(128→256, linear)
```

**Размер:** ~100 KB. Обучение: 10 000 шумовых примеров, 5 минут на GPU.

### Преимущества

- Метки не нужны совсем
- Ловит **любые** аномалии, включая неизвестные типы сигналов
- Порог θ настраивается на лету (адаптивный)
- Работает как второй уровень контроля поверх MLP

### Ограничения

- Не даёт координаты цели — только факт аномалии
- Может ложно срабатывать на редкие паттерны шума
- Нужна периодическая переобучение при смене условий шума

## 8. Гибридное объединение (Fusion)

### Весовая схема по SNR

```
# Адаптивные веса в зависимости от условий
if estimated_SNR > 15:
    w1, w2, w3 = 0.7, 0.2, 0.1   # доминирует Matched Filter
elif estimated_SNR > 7:
    w1, w2, w3 = 0.3, 0.6, 0.1   # доминирует MLP
else:
    w1, w2, w3 = 0.1, 0.5, 0.4   # MLP + Autoencoder (аномалия)

final_score = w1 * MF_score + w2 * MLP_prob + w3 * normalize(1/MSE)
```

### Логика принятия решения

```
final_score > threshold_high  →  УВЕРЕННАЯ ЦЕЛЬ → посылаем FM-m пучок
final_score > threshold_low   →  СЛАБЫЙ КАНДИДАТ → накапливаем следующие кубы
final_score < threshold_low   →  ШУМ → пропускаем
```

### Применение с перекрытием при инференсе

Так же как при обучении — кубы 16×16×16 с шагом 8 (overlap 50%):

```
Куб n:   z=[0..15]      → матрица E_n   → score_n
Куб n+1: z=[8..23]      → матрица E_{n+1} → score_{n+1}
...
```

Если цель на границе двух кубов — перекрытие гарантирует, что она попадёт
**целиком** хотя бы в один куб. Это ключевое преимущество overlap.

## 10. Схема обучения

```
ШАГИ ПОДГОТОВКИ:

1. Генерация датасета (GPU, ~10 мин)
   ├── 50 000 позитивов: цель в (kx,ky,kz) при SNR -5..25 dB
   ├── 30 000 негативов: чистый тепловой шум
   └── 20 000 помеховых: заградительные + кластерные помехи

2. Обучение Tiny MLP (GPU, ~20 мин)
   ├── Loss: BCE + 0.5×MSE_angle + 0.5×MSE_range
   ├── Optimizer: Adam, lr=1e-3
   └── Epochs: 50, batch=256

3. Обучение Autoencoder (GPU, ~5 мин)
   ├── Только на шуме (30 000 примеров)
   ├── Loss: MSE
   └── Epochs: 30, batch=256

4. Калибровка весов fusion (w1, w2, w3)
   ├── На валидационной выборке по SNR диапазонам
   └── Сохранение look-up таблицы весов

5. Экспорт весов в C-массивы для HIP кернела
   └── < 300 KB суммарно → помещается в константную память GPU
```

## 11. Выводы

**Пространство конечно:** 256 угловых бинов × ~10 SNR × ~8 условий = ~20 000 уникальных
паттернов. Датасет 50–100 K примеров полностью покрывает пространство — больше не нужно.

**Matched Filter** — основа при высоком SNR, нулевое обучение, полная интерпретируемость.

**Tiny MLP** — добавляет устойчивость при низком SNR и сложных помехах. Веса < 150 KB,
инференс < 0.1 мс на батче 1250.

**Autoencoder** — второй уровень контроля без меток. Ловит неизвестные аномалии,
которые MLP не видел при обучении.

**Overlap 50% при инференсе** критичен: гарантирует, что цель на границе кубов
попадёт целиком хотя бы в одно окно.

**Итоговое время coarse detection: ~0.85 мс** — весь пайплайн FFT + три ветви
классификатора + fusion на RX 9070.
