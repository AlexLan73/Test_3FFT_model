# Анализ: связь пика после дечирпа+FFT с дальностью (ось Z)

**Дата:** 2026-07-15 · **Контекст:** заявка СПОСОБ (гл. 2–3, `Doc/Patent/`), вопрос Alex:
«чистый один объект, по оси Z лежит пик — он связан с дальностью или нет?»

---

## 1. Короткий ответ

**Да, связан — линейно и однозначно.** Это не побочный эффект, а вся суть stretch-processing:
номер бина пика `k` = дальность в единицах разрешения:

```
R = k · Δr = k · c/(2·ΔF)        (при N_fft = N, неподвижная цель)
```

Оговорки (§4): доплеровское смещение, zero-padding, страддл, Найквист.

---

## 2. Вывод формулы (строго)

Излучаемый ЛЧМ (комплексная огибающая):
```
s_tx(t) = exp( j·2π·( f_c·t + μ·t²/2 ) ),      μ = ΔF / T_c
```

Эхо от точечной цели на дальности R: задержка `τ = 2R/c`, `s_rx(t) = s_tx(t − τ)`.

Дечирп — умножение на сопряжённую опору `s_ref*(t)`:
```
s_b(t) = s_rx(t)·s_ref*(t)
       = exp( −j·2π·( f_c·τ  +  μ·τ·t  −  μ·τ²/2 ) )
                 └─const──┘  └─ТОН f_b─┘ └──RVP───┘
```

Квадратичные члены `μt²/2` взаимно уничтожились — остался **чистый тон**:
```
f_b = μ·τ = μ·2R/c            — постоянен на всю длину записи
```
Формула в гл. 2.2 / заявке (`f_b = μ·2R/c`) — **верна**.

После FFT длиной `N_fft` при дискретизации `f_s` пик в бине:
```
k* = round( f_b · N_fft / f_s )   ⟹   R(k) = k · c·f_s / (2·μ·N_fft)
```
При `N_fft = N` и `T_c = N/f_s` подстановка даёт `R(k) = k·c/(2·ΔF) = k·Δr`.
**Шкала дальности — линейная, один бин = один Δr.**

## 3. Самопроверка на числах гл. 3.3

`ΔF = 6 МГц, f_s = 2 МГц, N = 10⁴` → `μ = 1.2·10⁹ Гц/с`, `Δr = 25 м`.
- бин 100 → R = 2.5 км; бин 4000 → 100 км.
- `R_max` (комплексная выборка, `f_b < f_s`) = `c·f_s/(2μ)` = **250 км** = `N·Δr` = 10⁴·25 м. ✅ Сходится — сетка бинов ровно покрывает однозначную дальность, параметры примера внутренне непротиворечивы.

## 4. Когда «связан, но…» — четыре оговорки

**4.1. Доплер (range–Doppler coupling) — главная.** Движущаяся цель добавляет `f_d = 2v_r/λ`:
```
f_b' = μ·2R/c ± f_d   ⟹   смещение по дальности  ΔR = v_r·T_c·f_c/ΔF
```
Числа для UHF-примера (`f_c = 0.5 ГГц, T_c = 5 мс, ΔF = 6 МГц`): **ΔR ≈ 0.42 м на 1 м/с**.
Цель 300 м/с → 125 м = **5 бинов** смещения. Это ровно то, что гл. 3.1/3.5 честно фиксирует:
«в одном импульсе дальность и доплер смешаны». Пик связан с дальностью, но **смещён
пропорционально радиальной скорости**; развязка — пачка (4-я ось, §3.5) либо up/down-чирп.

**4.2. Zero-padding.** Гл. 3.2 допускает добивку Z до степени двойки. Тогда шкала —
`R(k) = k·c·f_s/(2·μ·N_fft)`, где **N_fft — длина с нулями**, а не N. Физическое разрешение
остаётся `Δr = c/(2ΔF)` (его задаёт девиация), но «метров на бин» становится меньше.
⚠️ Ловушка реализации: пересчёт бин→метры обязан использовать `N_fft`.

**4.3. Страддл.** R непрерывна, сетка дискретна: пик между бинами → квантование ±Δr/2 и
scalloping-потеря до ~3.9 дБ (прямоугольное окно). Именно поэтому признаки токена
страддл-инвариантны — заявка тут самосогласованна. Окно Ханна (load-callback, гл. 3.4)
расширяет пик до ~2 бинов, **позицию не смещает** (окно симметрично).

**4.4. Найквист.** `f_b > f_s` (комплекс) → заворот спектра, пик ляжет в **ложный** бин.
Связь с дальностью однозначна только в пределах `R_max = c·f_s/(2μ)`.

**4.5. RVP (мелочь).** Остаточный член `exp(jπμτ²)` — чистая фаза: магнитудный пик не двигает,
на угловой FFT не влияет (константа по апертуре на данном бине). Важен лишь для тонкой
межимпульсной фазовой обработки; при необходимости убирается RVP-фильтром.

## 5. Следствие для арбитра (τ ≥ 0)

Линейность `f_b ↔ R` — **физический фундамент причинностного арбитра** заявки:
ретранслятор может только задержать (`τ_rep ≥ 0`) ⟹ ложная цель **всегда правее по Z**
истинной. Правило переднего края корректно ровно потому, что отображение
задержка→частота→бин монотонно и линейно. Единственная угроза правилу — не физика
дечирпа, а доплер-смещение (4.1): при разных `v_r` истинной и ложной целей на length-скоростях
порядок бинов сохраняется (125 м ≪ типичная задержка ретранслятора), но это стоит
упомянуть при защите.

## 6. Вердикт по тексту заявки

- `f_b = μ·2R/c`, `Δr = c/(2ΔF)`, отповедь `Δr = c/(2B)` — физически **верно**.
- «Дальность в Z ещё не проявлена — сидит в частоте биений» (гл. 3.1) — **верно**: до FFT ось Z = быстрое время, после FFT = дальность.
- «Одна цель → один тон на всю длину Z, посегментные окна не нужны» — **верно** (тон стационарен, глобальное FFT оптимально по разрешению).
- Рекомендация: в гл. 3.2/3.3 добавить одну фразу про пересчёт бин→метры при zero-padding (`N_fft` в знаменателе) и одну про доплер-смещение пика — снимет вопрос эксперта.
