## Что сделано

  - `angle_merge_tol=1`: джиттер-осколки `target` под НЕПОСРЕДСТВЕННЫМИ соседями (±1) заграда — поглощаются.

## ⚠️ Ограничение Варианта 1 (архитектурно, НЕ баг)

На ЧИСТОМ заграде остаётся ~93 ложных `target` (одиночные шумовые осколки под СЛУЧАЙНЫМИ углами,
разбросанные по всей карте — не ±1 соседи). Угловая толерантность гасит только прямых соседей.

**Это работа Варианта 2 (FM-m код), не Варианта 1.** По патенту §5.1 нельзя решать по яркости
(DRFM-копия может быть ярче), а по причинности τ≥0 одиночный осколок неотличим от слабой цели —
геометрия его давить не должна. §5.4: нужны ОБА варианта («геометрия ловит что упускает код, и
наоборот»). Осколок не ответит на свежий FM-m код → снимается Вариантом 2. Также §4.8: порог токена
МЯГКИЙ намеренно (передний край терять нельзя) — осколки плата за это.

## ✅ Вариант 2 — `CodeArbiter` (свежесть FM-m кода) — СДЕЛАНО (2026-07-18, Sonnet+ревью Кодо)

`arbiter.py` дополнен:
- `fm_correlate(ref, inp)` — numpy-коррелятор гл.6 §6.2: `IFFT(conj(FFT(ref))·FFT(inp))`, 2ⁿ, `|·|/N`.
- `_peak_to_floor_db` — пик/фон в дБ (RMS-фон). Эмпирика: свежий код ~30-38 дБ, шум/чужой ~11-12 дБ.
- **`CodeArbiter`** (§5.3): свежий код (пик≥20 дБ) → `target/fresh_code`; чужой/старый код → `false/
  stale_code`; неопрошенный угол → `target/no_probe` (§4.8); заград → `jammer`. Реюз `mseq.m_sequence_pow2`.
- **`CombinedArbiter`** (§5.4, Composite): геометрия И код — `comb/leading_edge` → target (геометрия
  сильна, код не нужен); одиночка + `stale_code` → **`false/cleared_by_code`** (ЧИСТКА осколков!);
  одиночка + `fresh_code` → `target/edge_and_code`; поглощённый джиттер + свежий код → `target/code_only`.

**Проверено Кодо независимо (smoke, degree=12):** цель (свежий код) 34.9 дБ→target; ретранслятор
(чужой) 11.6 дБ→false; осколок (шум) 12.2 дБ→false. CodeArbiter над смесью: цель→target, осколки→false.
test_arbiter 15 ok (Edge 6 + Code 6 + Combined 3), весь бэкенд `all_test.py` 0 fail.

**⚠️ Тонкость (важно для будущих тестов):** «чужой код ретранслятора» = НЕ другой `seed` той же
степени (разный seed = та же M-послед. со сдвигом фазы → коррелирует в пик!), а другой полином /
некоррелирующий ±1 код другого такта. §6.1: полином меняется по такту.

**Итог:** причинностный арбитр гл.5 замкнут — Вариант 1 (геометрия τ≥0, причинностные группы) +
Вариант 2 (свежесть FM-m кода, чистит одиночные осколки). Оба нужны (§5.4).
