## 1. Что зафиксировано (решения Alex 2026-07-18)

- **База = ex2**: 64×64×4096, fs=500 МГц, f_m=100 МГц, сцена 6 объектов (2 типа ×
  4/8/16 пер.), **дальность условная** (t0 в отсчётах), двухэтапный скан 32/32→16/8,
  полный `VolumeTokenizer` на тонком этапе, DataContext gen|disk, общая графика
  (`ScenePointsVisualizer`, дальность по горизонту).
- **S1 — ПРАВИЛЬНЫЙ сигнал объекта (главная правка)**: эхо = **задержанная копия
  зонда** `s(t − t0)`, а НЕ окно поверх глобальной волны. Для АМ это значит:
  **огибающая стартует с начала импульса** (фаза огибающей = 0 на фронте эха),
  для radio — фазовый сдвиг несущей (корректно само). Прецедент в core —
  `build_lfm_target_volume` (`waveform_to_cube.py:52`): «окно режет ЧУЖУЮ фазовую
  траекторию → размазанный пик; правильно — настоящее задержанное эхо».
- **S2 НЕ делаем**: длительности 4/8/16 остаются разными (дидактика), «один зонд
  на сцену» не требуем.
- **Помехи** — добавляются к правильным эхо (см. §3).

## 2. S1 — как реализуем (обсудить: правка задевает core)

Вариант A (рекомендую): маленькая функция в core рядом с `build_lfm_target_volume`
(тот же файл `waveform_to_cube.py` или `waveforms/echo.py`):

```python
def build_pulse_echo_volume(kind, f_c, n_units, t0_samples, kx, ky, ...) -> np.ndarray:
    """Эхо-копия зонда: импульс рендерится НА НУЛЕ (окно [0,dur], огибающая с фронта),
    затем сдвигается на t0 (np.roll без заворота) и раскладывается steering'ом.
    Реюз WaveformFactory + render_pipeline (формулы НЕ дублируем)."""
```

- АМ-зонд с «огибающей от фронта»: `AmWaveform` считает `1+m·cos(2π·f_env·t)` от t=0
  ⇒ рендер импульса НА t0=0 автоматически даёт огибающую с начала (cos=+1 на фронте),
  сдвиг готового куска на t0 сохраняет форму — ровно физика `s(t−t0)`.
- ex2 остаётся как есть (журнал прогресса); ex3 берёт новый генератор.
- Вариант B: та же функция на demo-уровне (core не трогаем). Минус: следующим
  примерам (ex4/ex5) и тактовому серверу она тоже понадобится — место ей в core.

## 8. 🔍 Глубокое ревью Кодо (правки сразу, ОБЯЗАТЕЛЬНО к учёту при реализации)

- **R1 (критично, эскалация ломается).** Заград/CW/VFD подсвечивают КАЖДОЕ грубое окно
  (полоса по всей оси на своём угле) ⇒ «всплеск везде» ⇒ ROI = вся ось ⇒ тонкий добор
  всей оси = минуты. ПОЭТОМУ порядок конвейера ЖЁСТКИЙ (он же сюжет «до/после»):
  1) грубый скан с помехой → **карта полосы** (окно×угол): угол помехи виден как
     столбец всплесков во всех окнах на одном (kx,ky) — это и есть barrage-признак
     ГРУБОГО уровня (сборка по окнам, дёшево, без токенизатора);
  2) **nuller** (`RobustMvdrNuller`) давит угол помехи на СЫРОМ объёме;
  3) повторный грубый скан → нормальные локальные ROI → тонкий добор → `VolumeTokenizer`
     → метрики объектов. Тонкий скан ДО nuller НЕ запускать при активной полосе
     (гейт: если всплесков > 1/3 окон — сразу к nuller).
- **R2 (время).** 6 помех × полный объём — дорого. Прогоны: базовый clean (без помех),
  базовый +10 дБ, затем 6×(помеха, SNR=+10). Итого 8 прогонов ≈ 5-7 мин — приемлемо,
  но в отчёте Sonnet замерить. Тесты (§6) — ТОЛЬКО barrage + DRFM на 16×16×512.
- **R3 (nuller контракт).** Прочитать `core/models/anti_barrage/mvdr.py` (сигнатуру
  `RobustMvdrNuller` — вход сырой объём/ковариация?) и `nuller.py` (SubspaceNuller) —
  НЕ гадать. Возможно, угол помехи для steering-вектора берётся из карты полосы п.R1-1.
- **R4 (ожидания по помехам).** Строгая приёмка только по двум: barrage → полоса/вердикт
  `barrage`; DRFM → `comb`. Остальные 4 (SMSP/INT_CW/IMP_ARC/VFD) — режим наблюдения:
  зафиксировать в отчёте/метриках, как их видит тракт (сигнатуры в кубе разные), БЕЗ
  жёстких ассертов — их классификация станет отдельной задачей.
- **R5 (JNR через амплитуду).** Помеха рендерится с `amplitude=10^(JNR/20)` (сигнал=1),
  `add_noise=False`, meta {nx,ny,kx,ky} — своя угловая позиция; суммируется комплексно.
- **R6 (эхо-генератор, ❗ семантика ex2 отличается).** Новая core-функция строит
  импульс НА НУЛЕ (`TimeWindow(short, t0=0, dur)`) → `np.roll`-сдвиг на t0 (без
  заворота: хвост за N обрезать нулями) → steering (реюз `render_pipeline` с
  `window=full` на уже сдвинутом 1D). Тест S1: огибающая АМ-эха на фронте = 1+m.
- **R7 (углы помех).** Не совпадать с объектами: barrage (+5,+18), DRFM (−22,−20),
  SMSP (+10,−8), INT_CW (−5,+25), IMP_ARC (0,−28), VFD (+28,−12).
