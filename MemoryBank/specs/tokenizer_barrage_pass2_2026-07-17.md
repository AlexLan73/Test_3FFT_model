# Находка: проход 2 токенизатора не ловит направленный barrage (2026-07-17)

> **Диагностика Кодо** на РЕАЛЬНОМ barrage-кубе (`BarrageRfJammer` → `LfmToCube`), не игрушечном.
> Переопределяет находку прошлого ревью («триаж smeared слабый») — корень глубже.

## Что показала диагностика

**Проход 1 (признаки по срезу) РАБОТАЕТ ПРАВИЛЬНО.** Реальный `BarrageRfJammer` с боресайта даёт в
каждом срезе признаки `source` (PR≈3.6, Hoyer≈0.94, MainFrac≈0.98) — НЕ `smeared`. Это физически
верно (патент §4-бис.4): направленный заград **собран по углу** (один источник-направление),
размазан по **дальности**. Метка `smeared` (размазанный по УГЛУ) для него неприменима — калибровать
smeared-порог НЕ нужно.

**Проход 2 (`assemble_range`) — БАГ.** На barrage-кубе (боресайт 0,0, заград во всех 1023 дальностях)
вердикты: `{target: 63, comb: 32, barrage: 1}`. Заград под (0,0) помечен **`target`** (не barrage),
и развалился на десятки ложных `target`/`comb` под соседними углами.

## Корень

1. **Угловой джиттер пика.** Из-за шума пик заграда в основном на (0,0), но иногда уезжает в
   соседний бин. Токены под (0,0) НЕ строго непрерывны по `r` (есть пропуски там, где пик уехал).
2. **Строгое условие `span==n` (contiguous)** в `_classify_group` слишком хрупкое: заград с
   редкими пропусками (fill≈0.88) не проходит barrage-ветку → fallback `target`.
3. **Джиттер-осколки** под соседними углами (по 5–50 токенов) дают ложные `target`/`comb`.

Патент §4.9: «токены во всех r подряд → заградка». Реальный заград = «почти во всех» → нужен
**fill-ratio**, не строгая непрерывность.

## План фикса (`core/models/tokenizer/tokenizer.py::assemble_range`/`_classify_group`)

1. **barrage по fill-ratio:** группа с `span >= min_barrage_run` И `n/span >= fill_threshold`
   (напр. 0.7) → `barrage`, ДАЖЕ с пропусками. Проверять ДО period-ветки (иначе почти-сплошной
   заград ложно ловится как comb).
2. **Различение barrage vs comb по fill:** barrage = высокий fill (>0.7, почти сплошь);
   comb = регулярный период с низким fill (<0.7). Сохранить существующую автокорр-проверку comb.
3. **Гашение джиттер-осколков:** мелкие неструктурные группы (мало токенов, не непрерывны,
   период не подтверждён) под НЕ-доминантными углами → не плодить ложные `target`. Порог по числу
   токенов группы или поглощение соседей доминантного barrage (±1 угловой бин).

**Сохранить:** `RangeAssemblyTests` (синтетика target/comb/barrage) зелёными. **Добавить:** тест на
РЕАЛЬНОМ barrage-кубе (боресайт → доминантный вердикт `barrage`, не `target`).
