- **Знание детектора (3.1):** НИЧЕГО, кроме: (1) шум — белый; (2) сигнал — один из
  обсуждаемых типов (radio/am). **Несущая НЕИЗВЕСТНА: может быть где угодно в
  {−250…+250} МГц** (уточнение Alex). Точные t0/длительность/тип — неизвестны.
  ⇒ Вариант A (точная опора) исключён; банк фиксированных полос — тоже.
- **Состав (3.2):** «два последних» = **C + D**, оба слепые по несущей:
  - **C (спектральный поиск полосы):** |FFT|² всей оси → шумовой пол → OS-CFAR-порог по
    спектру → найденная занятая полоса (где бы ни была в ±250) → полосовая маска → iFFT.
  - **D (Wiener):** `H = Ŝ/(Ŝ+N̂)` по оценённому спектру — полосу «находит» неявно.
  Оба — ФИЛЬТРЫ; детекционный хвост общий: |y|² → OS-CFAR(Pfa) по времени → кластеризация →
  найденные импульсы (t0, длительность). Плюс каждый вариант отдаёт оценку несущей f̂.
- **Pfa (3.3):** 1e-3.
> Вход: законченная ветка ex1 — ось N=4096 IQ, fs=500 МГц, 3 импульса (4/8/16 пер. f_m,
> t0=300/1600/2900), radio|am, f_m=250/100/50 МГц, белый гауссов шум SNR +20…−6 дБ.
> **Цель: НАЙТИ сигнал** (где на оси импульсы), а не «красиво почистить».

## 3. ВАРИАНТЫ (по нарастанию «слепоты» детектора)

### Вариант A — Согласованный фильтр (matched filter) 🏆 оптимум по SNR
- Опора = копия импульса из `WaveformFactory` (реюз, формулы не дублируем).
- `y = |correlate(x, ref)|` → пик = позиция импульса. Выигрыш ≈ 10·log10(D) дБ
  (D=длина импульса): для 160 отсч ≈ +22 дБ → находит даже при −6 дБ.
- Порог на y: `OsCfarDetector` (Pfa задаём честно, не «на глаз»).
- ➕ теоретический максимум, наша тема (радар и есть matched filter). ➖ нужна опора
  (тип+f_m+длительность известны); 3 длительности → банк из 3 опор.

### Вариант B — Энергетический детектор + CFAR (слепой)
- `p[n] = |x[n]|²` → скользящее окно → порог `OsCfarDetector` по оси времени
  → `DetectionClusterer` сливает соседние ячейки в импульсы.
- ➕ не знает про сигнал НИЧЕГО (только «энергия выше шумового пола»), 1 проход.
  ➖ проигрывает A ~на 10·log10(D)/2: на −6 дБ короткий импульс (8 отсч) не найдёт.

### Вариант C — Частотная фильтрация (полосовой FFT-фильтр) + B
- FFT всей оси → маска вокруг f_m (для am: f_m±f_env) → iFFT → дальше как B.
- Знает только f_m. Убирает шум ВНЕ полосы: выигрыш ≈ 10·log10(fs/полоса) ≈ +13…+17 дБ.
- ➕ середина по знанию/выигрышу; заодно даёт «очищенный сигнал» для картинки.
  ➖ прямоугольный импульс размазывается (Гиббс), границы t0 менее точные, чем у A.

### Вариант D — Wiener-фильтр по оценённому спектру
- Шумовой пол из `SpectrumSnrEstimator` → `H = S/(S+N)` → iFFT. Полу-слепой.
- ➕ красивое «очищение» без опоры. ➖ для ЦЕЛИ «найти» избыточен — после него всё
  равно нужен порог (B). Предлагаю НЕ брать в первую итерацию.

**Рекомендация Кодо: A + B в паре** (одна картинка: что видит оптимум vs слепой),
C — опционально третьим. D — отложить.
