## 0. Суть концепции (было → станет)

| | Было (фиксировано) | Станет (новая концепция) |
|---|---|---|
| Апертура | квадрат `n×n`, в частности `16×16` | прямоугольник **`i×j`** (число элементов по X и Y **независимо**) |
| Размер оси | 16 | каждая ось = **`2ⁿ`** (степень двойки) |
| Недобор элементов | — | реальное число < 2ⁿ → **добивается нулями** (zero-pad до ближайшей 2ⁿ) |
| Угловой FFT | `fft2` по факту размера | **паддит апертуру нулями до 2ⁿ** перед `fft2` (после окна) |
| Угл. шкала | `sinθ = k/8`, `k=−8…7` (при n=16, d=λ/2) | **по каждой оси своя**: `sinθ_x = k_x/(N_pad_x/2)`, `sinθ_y = k_y/(N_pad_y/2)` |
| Разрешение | `Δsinθ = 1/8` | `Δsinθ_x = 2/N_pad_x`, `Δsinθ_y = 2/N_pad_y` (у i≠j — **разное по X и Y**) |
| Диапазон k | `−8…7` | `k_x = −N_pad_x/2 … N_pad_x/2−1`, аналогично Y |
| Число ячеек | `M = 256` (=16²) | `M = N_pad_x · N_pad_y` |
| Куб | `16×16×L` | `i×j×L` |

**Физика паддинга:** zero-pad = интерполяция углового спектра. Разрешение определяет **реальная**
апертура (i, j), но бинов после FFT — `N_pad` (больше). Порядок в тракте: **окно (Хэмминг по реальным
элементам) → zero-pad до 2ⁿ → fft2 → fftshift**. Дальностная ось (`RangeConfig.n_fft`) уже так работает —
угловые оси зеркалят этот паттерн.

## 0-бис. Разграничение (НЕ путать при правке)

- **WMMA/MFMA-плитки 16×16 (и 32×32)** — аппаратная матричная инструкция GPU, **НЕ меняется**.
- **Логическая апертура i×j** (2ⁿ, zero-pad) — раскладывается ПЛИТОЧНО на нативные 16×16/32×32.
- В патентах гл.3 §3.4 и гл.4 §4.10 правка «16×16→i×j» **не должна** затрагивать описание железа.
- **Дальностная ось** (`n_real`/`n_fft`, `next_pow2` по дальности) — уже 2ⁿ-корректна, **ортогональна**
  этой задаче. Range-спеки (`range_scale_dictionary`, `LFM_dechirp_range_peak`) НЕ трогаем.

### 2.2. Тесты ДОБАВИТЬ под новую концепцию (обязательно)

1. **Неквадрат end-to-end**: `ArrayConfig(6,15)` → `vol.shape==(6,15,N)`, `ArrayGrid(6,15).steering().shape==(6,15)`.
2. **Zero-pad апертуры**: 6×15 физич. → padded (8,16); недостающие элементы = 0, ячеек = 8×16=128.
3. **Угловой FFT реально паддит** (headline): 6×15 → на выходе angular map **8×16**, `sinθ=k/(N_pad/2)` по оси.

4. **Угл. индекс на padded**: пик цели `(kx,ky)` → `round(kx)+N_pad_x//2`, `round(ky)+N_pad_y//2`.
5. **Коридоры признаков от M**: §4.11 масштабируется с `M=N_pad_x·N_pad_y` (не хардкод 128).
6. **squares/reduce_square = padded апертура** для неквадрата.
7. **Полный smoke 6×15→8×16**: `SceneServer.step` публикует cube `(6,15,·)` физич. + squares `(8,16)` padded.
