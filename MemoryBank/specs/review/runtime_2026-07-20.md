## Находки (severity ↓)

### [MED] ISP/LSP — `Transport` Protocol связывает publish+subscribe — transport.py:47,307
**Что:** `Transport(Protocol)` объявляет и `publish`, и `subscribe`. `WebSocketTransport.subscribe`
кидает `NotImplementedError` (publish-only шлюз для браузера) — подтип сужает контракт базового.
**Почему нарушение:** LSP (клиент, ждущий `Transport`, на `subscribe` получит исключение) + ISP
(publish-only шлюз вынужден «реализовать» ненужный метод заглушкой-исключением). Сейчас лечится
тем, что `FanOutTransport.subscribe` ловит `NotImplementedError` — это обход симптома.
**Как исправить:** разделить Protocol на роли — `PublishTransport` (`publish`) и `SubscribeTransport`
(`subscribe`); `ZmqTransport` реализует оба, `WebSocketTransport` — только publish. Тогда «publish-only»
выражено ТИПОМ, а не рантайм-исключением. (Обсудить: стоит ли, т.к. рабочий обход уже есть.)

### [MED] SRP/High Cohesion — `SceneServer` совмещает 5 ответственностей — scene_server.py:85
**Что:** `SceneServer.step()` делает: кинематику (project/propagate) + сборку объёма (VolumeBuilder) +
токенизацию + СЕРИАЛИЗАЦИЮ примитивов (`_tracks_payload`/`_jammers_payload`/`_tokens_payload`, ~60 строк) +
publish + приём команд. Сериализация «dataclass/numpy → примитивы» — отдельная ответственность.
**Почему нарушение:** SRP/High Cohesion — класс меняется по нескольким осям (изменился формат tick →
правим SceneServer; изменилась кинематика → тоже). Сериализаторы — кандидат в Pure Fabrication.
**Как исправить:** вынести `_*_payload` в отдельный `TrackSerializer`/`TokenSerializer` (Pure Fabrication),
инжектить в `SceneServer`. Тракт такта останется тонким координатором (Controller).
(Смягчение: докстринг класса честно объясняет, почему цикл тактов не переиспользует `MultiTactSequence` —
это осознанно; замечание только про сериализацию.)

### [LOW] `RawQueue.get` — одиночный `wait`, ложное пробуждение → преждевременный `None` — raw_queue.py:88-95
**Что:** `get(timeout)` делает один `wait(timeout)`; при spurious wakeup (или notify без данных) вернёт
`None` раньше реального таймаута.
**Почему:** для живого превью дроп/None приемлемы, но строго — не «ждал весь timeout».
**Как исправить:** цикл `while not self._frames: if not wait(remaining): return None` с учётом бюджета
времени. Низкий приоритет (consume-and-drop и так допускает потери).

### [LOW] `WebSocketTransport.publish` — побочный ленивый `start()` — transport.py:289-291
**Что:** `publish` при `self._thread is None` сам зовёт `start()`. Побочный эффект в публикации.
**Как исправить:** требовать явный `start()` (как в `PanelPublisher.start`) либо оставить, но
задокументировать как намеренную ленивость. Мелочь.

### [LOW] `codec`/`commands` — не глубоко покрыты этим ревью
`codec.py`/`commands.py` бегло: Command-паттерн + registry (`decode_command`) выглядят корректно,
VO-команды иммутабельны. Отдельный прогон при необходимости.

## Удачно
- **`PanelPublisher`** — чистый DI на `Transport` (Protocol), Facade над транспортом+`TickLog`;
  `start()` duck-типизированно регистрирует `on_connect` (не завязан на конкретный транспорт).
- **`Tick`/`RawFrame`** — корректные frozen VO (`slots=True`); `TickLog` под локом (гонка тред↔asyncio закрыта).
- **`RawQueue`** — consume-and-drop чётко реализован, `on_drop`-хук как точка расширения; `on_drop`
  вызывается ВНЕ лока (нет реентранси/дедлока).
- **`FanOutTransport`** — корректный Composite; **`Transport`** — узкий Strategy (2 глагола).
- **`RawCubeSource`/`FileSource`** — Strategy с чистым `iter_frames` (тестопригодность отделена от сети/тредов).
