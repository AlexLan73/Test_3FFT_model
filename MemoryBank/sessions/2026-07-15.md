## 🔑 Зафиксированные решения

- **Ветки ЛЧМ + АМ обе с старта. Схема FFT РАЗНАЯ (сверено с патентом, гл. 3 + 4-бис,
  «всё как в патенте, без упрощений»).** Вход обеих — `16×16 плоскостей × N по дальности`,
  **N — параметр 1024…10000** (16×16 = решётка nx×ny → kx×ky).
  - **ЛЧМ (точный):** 2 раздельных FFT — ① дальностный FFT **глобально по всей Z, 1 раз**
    (окна НЕТ), ② угловой FFT **16×16 поячеечно** на каждом бине → плотный куб `16×16×N`.
  - **АМ (грубый):** локальный **3D-FFT** по скользящему окну `16×16×D` (D=**16** дефолт,
    до **256**), **переменный шаг 8/16/32/64** → разреженные под-кубы.
  - Общее у веток — **куб-примитив `16×16×L` + объёмный токенизатор**, а НЕ FFT-этап.
    Плоская токенизация L=1 — частный случай.
  *(Исходное «АМ 16×16×16» и «окно/1250 кубов для обеих» — ошибка, поправлено 15:47 по
  патенту: окно/переменный шаг есть ТОЛЬКО у АМ; ЛЧМ = глобальный дальностный FFT.)*
- **Геометрия:** апертура 4…32 (не квадратная, zero-pad до 2ⁿ), Z 1024…10000.
- **Доплер:** 4-я ось (slow-time/такты, дефолт n_pulses=64) заложена, 4D-FFT — заглушка.
- **Движение:** cv + Марков-дрейф + широкий вираж; **без резких поворотов** (аэро-лимиты).
- **DataContext = базовый класс всего обмена данными** (централизованный I/O + шина
  Blackboard/Subject). Весь визуал — Observer, рисует только после помещения данных в шину.
- **Транспорт:** ZMQ PUB/SUB + MessagePack (нативный Observer, кросс py↔C++).
- **Инструменты:** Dear PyGui — live-2D-панель (реюз `PyPanelAntennas`: color_map/geometry/
  data-model), plotly/Three.js — 3D-обзор, matplotlib — контрольные PNG.
- **Цель:** поэтапно — точечная → протяжённая.
- **Квадраты:** Q2 заглушка + разворачиваемая закладка ±N плоскостей (блок на сигнал, 3 ряда).
- **Кросс-платформа:** сегодня Windows (numpy), завтра Linux (hip/rocFFT) — шина/панель/
  конфиг/протокол не меняются, меняется backend и источник данных.
