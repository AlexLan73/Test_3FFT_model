## 🧭 Схема по патенту (не отступать)

**ЛЧМ (гл.3, точный) — `LfmToCube`:**
0. ✅ **Дечирп (A9-gap2 — РЕШЕНО, вендорен 2026-07-15):** `core/generators/waveforms/heterodyne.py`
   → `dechirp(rx, ref) = rx·conj(ref)` (вендор DSP-GPU `HeterodyneDechirp` + GPUWorkLib `dechirp_numpy`).
   ⚠️ **Инъекция цели в P2 неверна для дечирпа:** сейчас цель = короткое окно (A9-gap1) → в сыром кубе
   размазана на 51 бин (637 м). **Правильно: цель = задержанное ЛЧМ-эхо** `getX_numpy(tau=+2R/c)` (полный
   чирп со сдвигом), тогда дечирп+range-FFT сжимают её в **~2 бина (50 м = Δr)**. Проверено (seed-демо
   `graphics/body_motion/p2_volume/dechirp_block.png`): R=5996 (истина 6000), длина 2 бина, угол точный.
   **P5 должен починить инъекцию цели** (delayed-chirp вместо short-window) в `VolumeBuilder`/`LfmToCube`.
   Окно Хэмминга **по дальности** давит боковые лепестки range-FFT; **по апертуре** — обязательно (§5).
1. **Дальностный FFT глобально по всей оси Z, ОДИН раз** (окна по дальности НЕТ — rect).
   ✅ **Рецепт DSP-GPU** (`heterodyne/t_heterodyne_step_by_step.py` step04 + `common/references/fft_refs.py`)
   + **словарь величин** [`specs/range_scale_dictionary_2026-07-15.md`](../specs/range_scale_dictionary_2026-07-15.md):
   zero-pad `N → N_fft=next_pow2(N)·(1|2|4)`, затем `np.fft.fft`. **ШКАЛА (единственная формула бин→метры):
   `R(k)=k·V1`, где `V1 = c·fs/(2·μ·N_fft)`** (зависит от N_fft! не путать с V3=c/2fs=12.5 м — это строб
   сырого фаст-тайма ДО дечирпа, после дечирпа НЕ применять). **Точечная цель = ОДИН всплеск**; его ширина
   = **Δr_eff = c/(2·μ·T_overlap)** (V2, от N_fft НЕ зависит — zero-pad только «прорисовывает»). Проверено:
   `graphics/body_motion/p2_volume/range_profile.png` (R=6002 при истине 6000, ширина ~44 м = Δr_eff при 53% перекрытия).
   ⚠️ **Знак:** `dechirp=rx·conj(ref)` → `f_b=−μτ` → **зафиксировать ОДНУ конвенцию** (маппить по `|f|`
   ЛИБО `ref·conj(rx)`). Данные держим **комплексными** (магнитуда только после углового FFT).
2. **Угловой FFT 16×16 поячеечно** — на **каждом** бине дальности (реюз углового ядра `fft3d`).
3. `|·|²` **только после** углового FFT. Выход — плотный куб `16×16×N` (угол×угол×дальность).

**АМ (гл.4-бис, грубый) — `AmToCube`:**
1. **Локальный 3D-FFT** по скользящему окну `16×16×D` (D=16 дефолт, до 256).
2. **Переменный шаг 8/16/32/64**, под-кубы **внахлёст ½** (шаг=D/2 дефолт: D16→шаг8, D256→шаг128).
   Возможен **произвольный под-куб** (любой размер/позиция, «только конец») — заложить параметрами.
3. Выход — разреженные под-кубы `16×16×D`.

## ✅ Критерии приёмки
- **ЛЧМ:** дальностный FFT — **один глобальный** по Z (не посегментный); угловой — поячеечно;
  `|·|²` после углового. Точечная цель → пик в `(kx,ky)` на своём бине дальности.
- 🎯 **СЖАТИЕ ПО ДАЛЬНОСТИ — критерии из словаря** (`range_scale_dictionary_2026-07-15.md` §4):
  - `|R_est − R_true| ≤ V1/2` (полбина шкалы `V1=c·fs/2μN_fft`);
  - ширина всплеска −3дБ = **`c/(2·μ·T_overlap)` ± 20%** (rect, без окна по дальности); сравнивать
    **с Δr_eff, НЕ с номиналом** c/2ΔF (номинал 25 м достигается лишь при полном перекрытии τ≪T);
  - **инвариант zero-pad:** ×4 меняет `V1` в 4 раза, а **ширину в метрах НЕ меняет** (проверить);
  - точечная цель = **1 окно закладки ±N** (2 на границе), НЕ 5. Сырой 51-бин (637 м) — длина
    импульса ДО дечирпа, НЕ цели. Сигнатуры: точка=1 окно · протяжённая=2–3 · гребёнка=разнесённые
    точки · заград=во всех окнах (полоса). Окно Хэмминга по дальности **не ставить до измерения ширины**
    (уширяет ×~1.5).
- **АМ:** локальный 3D-FFT по окну `16×16×D`; шаг 8/16/32/64 меняет нахлёст/плотность;
  компактный источник → колокол, непрерывный → размазанный профиль (концентрация как признак).
- Выход обеих сводится к кубу/квадрату 16×16 (общий низ); имя 3-й оси — range.
- Сверка ЛЧМ↔эталон (numpy): пик по дальности = `R(state)` ± бин; углы = `(kx,ky)` ± бин.
