# Шаг 2 — Кернел B: угловой FFT + магнитуда + признаки + ИИ-триаж

## Фьюзед WMMA/LDS кернел грубого обнаружения

**Дата:** 07.07.2026
**Платформа:** AMD RX 9070 (RDNA4/gfx1201) · ROCm 7.2 · HIP + WMMA
**Вход:** комплексный буфер `16×16×10000` из Кернела A (Z = дальность)

---

## 1. Что делает Кернел B

Один workgroup обрабатывает 16 бинов дальности, куб живёт в LDS, наружу уходит только токен:

```
fused kernel (workgroup = 16 бинов дальности):
  1. WMMA: угловой FFT 16×16 (комплекс)              ← фаза ещё нужна
  2. |·|²  → карта энергии 16×16                     ← магнитуда, переход в float
  3. линейка признаков по карте (редукции в LDS)
  4. MLP-триаж на признаках → метка + скор
  5. на выход: токен (только выжившие)
```

Магнитуда берётся здесь, а не раньше: угловой FFT — когерентное сложение по апертуре, до него держим комплекс.

---

## 2. Линейка признаков (256 → ~6)

Детектор кормим НЕ сырой картой 16×16 (256 входов → тяжёлая сеть), а вектором инженерных признаков. Легче сеть, интерпретируемо, сертифицируемо. Признаки — дешёвые редукции в том же LDS. `p_i = |·|²`, `n = 256`.

| Признак | Формула | Что ловит |
|---|---|---|
| PR | `(Σp)² / Σp²` | собранность (источник vs шум) |
| Hoyer | `(√n − ℓ₁/ℓ₂)/(√n − 1)` | разреженность, [0,1] |
| max/mean | пик к среднему | яркость пика |
| позиция пика | `argmax (kx,ky)` | направление |
| 2-й/1-й пик | отношение вершин | одиночный vs множественный |
| энергия | `Σp` | абсолютный уровень |

Статус: закладываем все 6, по результатам уберём лишние. Кандидаты на добавление позже: энтропия карты, число пиков выше порога.

Все считаются за 1–2 прохода по 256 значениям, аккумуляторы в LDS.

---

## 3. ИИ-триаж (детектор классов)

Крошечный MLP: `6 входов → 16 → K классов`, веса единицы КБ, влезают в LDS рядом с картой.

Классы — закладываем 4, не пойдёт → свернём до 3:

```
шум  /  одиночный  /  заградка  /  гребёнка-DRFM
```

Выход: метка + скор (вероятность класса).

---

## 4. Принципиальная оговорка: триаж — гейт, не арбитр

Детектор **приоритизирует**, но не выносит финальный вердикт «цель». Причина: DRFM даёт «одиночный чистый пик», для угловой сети неотличимый от настоящей цели. Поэтому:

- выход детектора = `скор` в токене → по нему приоритет и порядок обработки;
- метку «цель» окончательно ставит **физика на L2** (передний край + текущий M-код), а не эта сеть.

Это требование защиты от адаптивного противника, не опция.

---

## 5. Токен на выходе

Только выжившие (прошедшие гейт):

```
токен = (kx*, ky*, n_дальности, метка_класса, скор)
```

Пустые/шумовые ячейки не пишутся → разрежённый выход.

---

## 6. Обучение триажа (набросок, детализируем позже)

- синтетический генератор сцен на GPU: цель, гребёнка-DRFM, заградка, RFI, чистый шум, разные SNR;
- метки бесплатны (сцена известна);
- вход сети — вектор из 6 признаков, посчитанных тем же кодом, что в кернеле (одна реализация признаков для train и inference).

---

## 7. Оценка времени Кернела B

Считается после фиксации: число признаков, размер MLP, стоимость `|·|²` и WMMA-FFT. Складывается с Кернелом A (окно+FFT) в полный бюджет `16×16×10000`. Замер на gfx1201 — отдельным прогоном.

---

## 8. Открытые вопросы к Шагу 2

1. Признаки — стартовый набор 6 (правим по результатам).
2. Классы — 4 (шум/одиночный/заградка/гребёнка), при неустойчивости → 3.
3. Порог гейта — по скору или по PR отдельно (ранний выход до MLP на явном шуме).
