# Единое решение: каскадный ECCM-детектор на 3D-данных ФАР

## План реализации и техническое описание (v1)

**Дата:** 07.07.2026
**Платформа:** AMD RX 9070 (RDNA4/gfx1201, 16 GB VRAM) · ROCm 7.2 · HIP · rocFFT/hipFFT
**Приоритеты (в этом порядке):** надёжность обнаружения → скорость обнаружения → красота решения

---

## 0. Принципы решения

Три сквозных принципа, из которых выводится вся архитектура:

1. **Дешёвый инвариант гейтит дорогой зонд.** Сначала считаем дешёвые
   статистики, устойчивые к противнику (концентрация, передний край), и только
   на выживших тратим дорогой и агильный FM+M зонд и глубокий анализ.
2. **Физика — арбитр, нейросеть — адъюнкт.** Окончательное решение
   «цель/помеха» принимает детерминированный физический признак (передний край
   + корреляция с текущим/историческим M-кодом + задержка петли). Нейросеть
   ускоряет приоритизацию, но не может переопределить физику. Это требование
   безопасности против адаптивного противника, а не опция.
3. **Каждый уровень отвечает на вопрос, который предыдущий не может.**
   L1 — «где собрана энергия», L2 — «кто из пиков настоящий», L3 — «что это за
   сигнал». Никакого дублирования работы между уровнями.

## 1. Геометрия данных `16×16×10000`

Вход после гетеродина — комплексный тензор I/Q:

| Ось | Размер | Смысл |
|-----|--------|-------|
| X | 16 | приёмная апертура (азимут) |
| Y | 16 | приёмная апертура (угол места) |
| Z | 10000 | быстрое время (дальность) |

### 1.1 Нарезка по дальности

Окно `W = 16`, шаг `s = 8` (overlap 50%):

```
N_frames = floor((N_Z - W)/s) + 1 = floor((10000 - 16)/8) + 1 = 1248 + 1 = 1249
```

**Внимание:** точное значение — **1249**, не 1250 (частая ошибка округления).
Overlap 50% гарантирует, что всплеск не потеряется на границе окон и целиком
попадёт хотя бы в одно окно.

### 1.2 Параметры, которые надо зафиксировать (влияют на всю математику)

- **Канал.** ЛЧМ (дечирпованный), M-последовательность, или совмещённый?
  От этого зависит, даёт ли FFT по быстрому времени дальность (дечирп ЛЧМ:
  биения ↔ дальность) или дальность приходит из корреляции (M-код).
- **Структура по Z.** Один импульс (только дальность) или пачка (дальность +
  доплер по медленному времени)? В одном снимке `16×16×10000` доплера
  физически нет — он живёт между импульсами. Это определяет смысл оси `k_z`.
- **Шаг решётки и λ.** UHF 0.5 ГГц → λ ≈ 0.6 м. При `d = λ/2 = 0.3 м`
  отображение бина в угол: `sinθ_m = m·λ/(N·d) = m/8` (N=16). Подтвердить `d`.
- **Частота дискретизации `f_s`.** Даёт метр на отсчёт `Δr = c/(2 f_s)` и
  однозначную дальность `10000·Δr`. Нужна для метрической привязки.

### 1.3 Отображение бинов в физические величины

Угол прихода (direction cosines), после `fftshift`, `m ∈ [-8, 7]`:

```
sinθ_az = k_x · λ / (N_x · d_x)      → при d=λ/2:  sinθ_az = k_x / 8
sinθ_el = k_y · λ / (N_y · d_y)      → при d=λ/2:  sinθ_el = k_y / 8
```

Разрешение по sinθ ≈ `2/N = 1/8` у оси визирования (≈7° у boresight, грубее к
краям) — достаточно для целеуказания пучка FM+M.

Грубая дальность = номер окна: `R_coarse ≈ n*·s·Δr` (разрешение = шаг `s=8`).
Тонкая дальность добирается на L2 внутри окна.

## 2. Общая схема каскада

```
16×16×10000
  → нарезка (окно 16, шаг 8) → 1249 окон
  → L1  грубое обнаружение (ВСЕ 1249, один fused WMMA/LDS кернел)
        2D угловой FFT (оконный) + энергия вдоль окна
        статистика концентрации (PR/Hoyer) + локальный OS-CFAR по углу
  → 16×16×1249 разрежённая карта → кандидаты (дедуп/NMS, потолок K_max)
  → L2  дискриминант DRFM (только кандидаты, ~50–100)
        k_z-FFT (тонкая дальность) + окна
        передний край + M-код (текущий/исторический) + задержка петли
        НН-триаж как адъюнкт; физика — арбитр
  → L3  глубокий добор (единицы, ~5)
        код-корреляция > MUSIC/ESPRIT под когерентной DRFM; тип сигнала
  → FM+M пучок (8–10 лучей, конус ±1–2 угловых бина вокруг R_coarse)
```

---

## 3. L1 — грубое обнаружение (все 1249 окон)

### 3.1 Ключевое решение: НЕ 3D-FFT, а 2D угловой FFT + энергия

По теореме Парсеваля `Σ_kz |F(k_x,k_y,k_z)|² = W · Σ_z |f(k_x,k_y,z)|²`.
Значит суммирование по `k_z` = энергия во времени, и её можно взять **без FFT по
этой оси**. Полный 3D-FFT с последующим схлопыванием `k_z` — выброшенная работа
(≈треть флопов) на всех 1249 окнах. Ось `k_z` (тонкая дальность) нужна только на
кандидатах → переносим её FFT на L2.

L1 = 2 прохода углового FFT (`k_x`, `k_y`) + аккумуляция энергии вдоль окна.
Оконная функция (Хэмминг/Хэнн) применяется **уже на L1** — 16-точечный FFT
прямоугольным окном даёт боковики ≈ −13 дБ, и сильная помеха течёт в соседние
угловые/дальностные ячейки. На грубом этапе обнаружение важнее разрешения →
окно оправдано.

### 3.2 Статистика обнаружения — концентрация, не суммарная энергия

Суммарная энергия по углу `E_n = Σ_{kx,ky}` НЕ годится: собранная цель и
размазанная заградка с равной энергией неразличимы. Меряем **концентрацию**
по 256 значениям угловой карты `p_i = |x_i|²`:

- **Participation Ratio** (эффективное число зажжённых ячеек):
  `PR = (Σ p_i)² / Σ p_i²`. `PR ≈ 1–3` → собрано (источник); `PR ≈ 256` → шум.
  Две суммы-редукции, без сортировки — идеально для LDS.
- **Hoyer** (ратио-форма, [0,1]): `H = (√n − ℓ1/ℓ2)/(√n − 1)`, `n=256`.

Эти статистики дёшевы (пара аккумуляторов) и переживают заградку/RFI, у которых
высокая суммарная энергия, но низкая собранность.

### 3.3 Порог — локальный устойчивый CFAR, не глобальный

Глобальный CFAR по всем 1249 окнам ломается под самоприкрытием: гребёнка DRFM и
непрерывный источник («радиолюбитель», заполняет всю дальность) задирают
`μ, σ` → self-masking → истинная (возможно слабая) цель уходит под порог.

- **OS-CFAR** (order-statistic): порог = `α · x_(k)` по отсортированным
  референсным ячейкам — устойчив к нескольким интерферентам в окне.
- **По-угловой**: CFAR в ячейках угловой карты (RFI локализован по углу), а не
  по угло-суммарному скаляру.
- Локальное скользящее окно по дальности с guard-ячейками.

### 3.4 Выход L1: разрежённая карта `16×16×1249`

После гейта в каждом срезе горят 0–3 угловые ячейки. Карта почти вся ноль.
Кандидат = токен `(k_x, k_y, n, скор)`. Далее:

- **дедуп/NMS**: overlap 50% → одна цель даёт 2 соседних окна → слить;
  плюс overlap коррелирует референсные ячейки CFAR (занижает σ) → учитывать
  (брать несмежные окна для оценки шума).
- **потолок K_max + приоритет по скору**: защита от затопления кандидатами
  (см. §7). Фиксированный бюджет L2/L3 → ограниченная латентность.

### 3.5 Реализация: fused WMMA/LDS кернел

Один workgroup = одно окно. Куб/срез живёт в LDS, наружу уходит только токен.
Угловой DFT-16 = матумножение на матричных ядрах (v_wmma). Энергия копится в
эпилоге того же кернела в LDS (нет 3D-спектра в VRAM → меньше трафика, выше
occupancy). PR/Hoyer — редукция там же. Ранний выход: если PR говорит «размазано»
— пишем «пусто» и не тратим свёртку.

**Стоимость L1:** ~2/3 от 3D-версии по FFT + дешёвая редукция; батч 1249.

## 4. L2 — дискриминант DRFM (только кандидаты, ~50–100)

Здесь ось `k_z` оправдана: тратим FFT по быстрому времени → **тонкая дальность**
внутри окна (расширенный патч 16×16×32 с окнами Чебышёв/Хэмминг по апертуре,
Хэнн/Блэкман по дальности; края clamp'ить: `z_start = max(0, n*·8 − 8)`).

**Физическое ядро (то, чего нет у generic-детектора):**

1. **Правило переднего края.** DRFM причинно умеет только задерживать →
   ложные цели **никогда не ближе** истинной. Истинная = **ближний** пик в
   группе, не самый яркий. `R_true = min` дальность в группе пиков.
2. **Корреляция с текущим/историческим M-кодом.** Пер-импульсный агильный код
   (M-последовательность через ЦАП). Отклик на **текущем** коде = настоящая
   цель; отклик на **устаревшем** коде = репитер.
3. **Задержка петли.** По тому, какой исторический код совпал, оценивается
   латентность петли DRFM → дополнительный структурный дискриминант и признак
   типа джаммера.

Нейросеть на этом уровне — быстрый триаж/приоритизация, **адъюнкт**. Она не
может пропустить ближний слабый пик или переопределить корреляционный признак:
физика — арбитр. Причина — устойчивость к адаптивному противнику: MF согласован
с кодом и потому идеально коррелирует с **копией** DRFM (тот же код); AE ловит
«не-шум» одинаково у истинной и ложной; MUSIC ломается на когерентных
источниках. Все ветки generic-ML побеждаются самоприкрытием — спасает только
причинность переднего края и агильность кода.

**Выход L2:** `n*`, `k_z*` (тонкая дальность), `(k_x*, k_y*)`, метка
цель/репитер/заградка, оценка латентности петли.

---

## 5. L3 — глубокий добор (единицы, ~5)

Длинный фрагмент `16×16×256` вокруг грубой дальности:

- **код-корреляция как основной инструмент** точной дальности/подтверждения
  (устойчива к когерентной копии);
- MUSIC/ESPRIT — только для **доброкачественного** многоцелевого уточнения
  углов; под когерентной DRFM требуют пространственного сглаживания →
  съедают раскрыв 16×16 → применять осторожно;
- анализ огибающей → тип сигнала (SMSP-размытие спектра, ISRJ-гребёнка и т.д.);
- тяжёлый уровень (5–10 мс/фрагмент), запускается редко → не доминирует, **при
  условии** ограничения числа кандидатов (§7).

---

## 6. Связь с FM+M пучком

Из L2/L3 на каждый подтверждённый кандидат: `n*`, `k_z*`, `(k_x*, k_y*)`.
Формируем пучок 8–10 FM+M лучей, покрывающий конус неопределённости по углу
(±1–2 бина) и диапазон дальности вокруг `R_coarse`. Когнитивная петля:
обнаружил (пассивно) → зондировал (пучком) → уточнил. Экономия эфира — гейт
активного зонда дешёвыми инвариантами L1–L2 (пункт новизны патента).

---

## 7. Защита от противника и латентность

- **Анти-затопление.** Джаммер, зажигающий сотни окон, взрывает L2/L3. Жёсткий
  `K_max` кандидатов + приоритет по инвариантному скору (концентрация +
  передний край) + мягкая деградация. Фиксированный бюджет = **детерминированная
  латентность** (требование реалтайма).
- **Устойчивый CFAR** (OS, локальный, по-угловой) — см. §3.3.
- **Окна на всех FFT-этапах** — против утечки коротких преобразований.
- **Физический якорь неподделываем**: противник может подтолкнуть НН-скор, но
  не причинность переднего края и не текущий код.

## 8. Что исправлено против версии-конкурента

| Место | Было (generic-каскад) | Стало (это решение) |
|-------|----------------------|---------------------|
| L1 FFT | 3D-FFT 16³ + схлопывание k_z | 2D угловой FFT + энергия (Парсеваль) |
| Ось k_z | схлопнута И же argmax'ится (противоречиво) | определена как тонкая дальность, живёт на L2 |
| Статистика L1 | угло-**суммарная** энергия | концентрация (PR/Hoyer) |
| Порог | глобальный CFAR по 1249 | локальный OS-CFAR по углу |
| Дискриминация | generic MF/CNN/AE/MUSIC | физическое ядро (край+код+латентность) |
| Fusion | SNR-веса (порочный круг, смещён к MF) | физика-арбитр, НН-адъюнкт |
| Латентность | без потолка (DoS-уязвим) | K_max + приоритет + деградация |
| Окна | только на L2 | на всех FFT, включая L1 |
| Мелочи | 1250 (off-by-one), z_start<0 | 1249, clamp краёв |

## 9. Дорожная карта реализации (по шагам)

Идём по шагам, каждый — с математикой и расчётами:

- **Шаг 1.** Данные `16×16×10000`: зафиксировать канал, структуру по Z, `f_s`,
  `d/λ`; посчитать `Δr`, однозначную дальность, отображение бин→угол. Собрать
  генератор тест-сцены (цель + DRFM-гребёнка + заградка + RFI).
- **Шаг 2.** L1-ядро: оконный 2D угловой FFT + энергия; вывести и проверить
  PR/Hoyer на тест-сценах; калибровка OS-CFAR (Pfa).
- **Шаг 3.** Разрежённая карта + дедуп/NMS + K_max; метрики Pd/Pfa/латентность.
- **Шаг 4.** L2: патч 16×16×32, тонкая дальность, правило переднего края;
  интеграция M-коррелятора (текущий/исторический код, degree-31 LFSR).
- **Шаг 5.** Задержка петли; НН-триаж как адъюнкт; проверка «физика-арбитр».
- **Шаг 6.** L3 + связь с FM+M пучком; сквозная латентность на потоке.
- **Шаг 7.** Fused WMMA/LDS кернел L1; замер на gfx1201.
