# Шаг 2 — Кернел B: угловой FFT + магнитуда + признаки + ИИ-триаж

## Фьюзед WMMA/LDS кернел грубого обнаружения + токен со структурой

**Дата:** 07.07.2026
**Платформа:** AMD RX 9070 (RDNA4/gfx1201) · ROCm 7.2 · HIP + WMMA
**Вход:** комплексный буфер `16×16×10000` из Кернела A (Z = дальность)

---

## 1. Что делает Кернел B

Один workgroup обрабатывает 16 бинов дальности, куб живёт в LDS, наружу уходит только токен:

```
fused kernel (workgroup = 16 бинов дальности):
  1. WMMA: угловой FFT 16×16 (комплекс)          ← фаза ещё нужна
  2. |·|²  → карта энергии 16×16                  ← магнитуда, переход в float
  3. линейка признаков по карте (редукции в LDS)
  4. поиск пиков по углу (до 5)
  5. MLP-триаж на признаках → метка среза + скор
  6. на выход: токен со структурой (только выжившие)
```

Магнитуда берётся здесь, а не раньше: угловой FFT — когерентное сложение по апертуре, до него держим комплекс.

---

## 2. Две ортогональные оси структуры

Ключевая развязка ролей:

- **по УГЛУ** — несколько источников под разными направлениями на ОДНОЙ дальности → ловит **массив пиков** в токене (проход 1);
- **по ДАЛЬНОСТИ** — один источник, несколько откликов вдоль Z (гребёнка репитера) → ловит **сборка токенов** вдоль Z (проход 2).

Механизмы не пересекаются: массив = много целей по углу; гребёнка = проход 2 по дальности.

---

## 3. Линейка признаков (256 → ~6)

Детектор кормим НЕ сырой картой 16×16 (256 входов → тяжёлая сеть), а вектором инженерных признаков. Легче сеть, интерпретируемо, сертифицируемо. Признаки — дешёвые редукции в том же LDS. `p_i = |·|²`, `n = 256`.

| Признак | Формула | Что ловит |
|---|---|---|
| PR | `(Σp)² / Σp²` | собранность (источник vs шум) |
| Hoyer | `(√n − ℓ₁/ℓ₂)/(√n − 1)` | разреженность, [0,1] |
| max/mean | пик к среднему | яркость пика |
| позиция пика | `argmax (kx,ky)` | направление |
| 2-й/1-й пик | отношение вершин | одиночный vs множественный |
| энергия | `Σp` | абсолютный уровень |

Статус: закладываем все 6, по результатам уберём лишние. Кандидаты на добавление позже: энтропия карты, число пиков выше порога. Все считаются за 1–2 прохода по 256 значениям, аккумуляторы в LDS.

---

## 4. Токен со структурой (массив пиков до 5)

На одном срезе может быть несколько пиков по углу → токен это не одна точка, а список фиксированной длины (GPU: нет динамической памяти, массив в регистрах/LDS):

```
токен_среза = {
    n_дальности,
    n_пиков,                         // сколько реально нашли (0..5)
    пики[5] = {                      // массив фиксированной длины, с запасом
        kx, ky,                      // направление пика
        яркость,                     // амплитуда
        кромка                       // нарастание/спад по соседним бинам Z
    },
    PR, Hoyer, скор, метка_среза     // общие по срезу
}
```

- `n_пиков` — сколько из 5 валидны;
- если пиков > 5 (сильная заградка) — это само по себе признак «размазано» → метим срез как заградку, лишние не пишем;
- **кромка** (передний фронт пика по дальности) — почти бесплатна (соседние бины Z уже в LDS), даёт: (а) частичный передний край уже на проходе 1; (б) физпризнак «резкая кромка реального отражателя vs размытая копия DRFM», который трудно подделать.

---

## 5. ИИ-триаж: 3 честные метки на срезе

Крошечный MLP: `6 входов → 16 → 3 класса`, веса единицы КБ, в LDS рядом с картой.

Метки **на срезе** (то, что физически видно по одной угловой карте):

```
шум  /  собранный источник  /  размазанный (заградка)
```

4-й класс (гребёнка-DRFM) на срезе НЕ определяется — он отличается от цели по дальности, а не по углу. Его выносит проход 2.

Порог записи токена — **мягкий по скору**: слабый ближний пик (истинная цель под яркой гребёнкой) не должен отсеяться, передний край терять нельзя. Разрежённость сохраняется, надёжность растёт.

---

## 6. Триаж — гейт, не арбитр

Детектор **приоритизирует**, но не выносит вердикт «цель». DRFM даёт «собранный источник», для угловой сети неотличимый от настоящей цели. Поэтому:

- выход детектора = метка среза + скор → приоритет и порядок обработки;
- ярлык «цель / DRFM» ставит проход 2 (по дальности) и физика L2 (передний край + текущий M-код), не эта сеть.

Требование защиты от адаптивного противника, не опция.

---

## 7. Проход 2 — сборка структуры вдоль дальности

Собираем токены с меткой «собранный» под одним углом `(kx*, ky*)`, раскладываем по `n_дальности`:

| Картина токенов вдоль Z | Вывод |
|---|---|
| один токен | одиночная цель |
| несколько с равным шагом Δn | гребёнка (репитер); Δn = задержка петли DRFM |
| ближний токен группы | передний край = истинная цель (DRFM только задерживает) |
| токены во всех n подряд | заградка / RFI (сплошняк, не гребёнка) |

Обработка — дешёвая, единицы токенов, не куб: период ищется автокорреляцией цепочки, передний край — минимальным `n`. Это и есть гейт для FM+M.

**Итог по классам: 4 класса живут, но на разных уровнях:**
- срез (проход 1): шум / собранный / размазанный (видно по углу);
- профиль (проход 2): собранный распадается на цель / гребёнку-DRFM (по переднему краю + периоду вдоль Z).

## 8. Вариант 2 — патч + 3D-CNN (L3-добор, НЕ основной путь)

Для нерегулярной структуры, которую автокорреляция цепочки токенов не берёт: рваная гребёнка, ISRJ с дрожащей задержкой, перекрытие двух джаммеров.

Важно (ограничение):

- структура живёт по дальности → патч тянем по Z: `16×16×64` / `128` / `256`;
- это **не куб в LDS** (`16×16×256` = 512 КБ, в 32 КБ LDS не лезет) → **отдельный тяжёлый кернел с чтением из VRAM**, не фьюзед;
- значит вариант 2 = **L3-инструмент** (глубокий добор, редкий), запускается только на кандидатах, которых проход 2 пометил «структура есть, но не регулярная».

**Открытый вопрос (думать и тестировать):** глубина патча `64 / 128 / 256` зависит от типичного периода гребёнки (число бинов между ложными). Пока период неизвестен — глубину не фиксируем. Обучение 3D-CNN на такой глубине тяжелее фьюзед-триажа → тем более не в основном потоке.
