## Суть изменения

| Было (в тексте глав) | Стало (новая концепция) |
|---|---|
| квадратная апертура `n×n`, в частности `16×16` | прямоугольная **`i×j`** — число элементов по X и Y **независимо** |
| размер оси = 16 | каждая ось = **`2ⁿ`** (степень двойки) |
| фиксированное число элементов | реальное число < 2ⁿ → **добивается нулями** (zero-pad до ближайшей 2ⁿ) |
| `sinθ = k·λ/(N_ап·d) = k/8`, `k=−8…7` | по каждой оси своя: **`sinθ_x = k_x/(N_pad_x/2)`**, **`sinθ_y = k_y/(N_pad_y/2)`** |
| разрешение `Δsinθ = 1/8` | `Δsinθ_x = 2/N_pad_x`, `Δsinθ_y = 2/N_pad_y` — при `i≠j` **разное по X и Y** |
| диапазон `k = −8…7` | `k_x = −N_pad_x/2 … N_pad_x/2−1`, аналогично Y |
| число угловых ячеек `M = 256` | **`M = N_pad_x · N_pad_y`** |
| куб `16×16×L`, тензор `16×16×N` | `i×j×L`, `i×j×N` |
| «256 антенн / 256 элементов» | `i·j` каналов |
| шум `PR ≈ M/2 = 128` | `PR ≈ M/2`, где `M = N_pad_x·N_pad_y` |

## Два важных разграничения (НЕ путать при переработке)

1. **WMMA/MFMA-плитки `16×16` (и `32×32`)** — это аппаратная матричная инструкция GPU, она
   **НЕ меняется**. Логическая апертура `i×j` (2ⁿ, zero-pad) раскладывается ПЛИТОЧНО на нативные
   16×16/32×32. В §3.4 (гл.3) и §4.10 (гл.4) правку «16×16→i×j» вести так, чтобы описание железа
   осталось прежним, а логическая апертура стала i×j.
2. **Дальностная ось** (быстрое время, `N`, добивка до 2ⁿ по дальности) — уже описана верно и
   **ортогональна** этому изменению. Трогаем только **угловые** оси (X, Y апертуры).

## Физика паддинга (одинаково для ЛЧМ- и AM-фронтенда)

Порядок: **окно (Хэмминг по РЕАЛЬНЫМ i×j элементам) → zero-pad до 2ⁿ → угловой FFT → fftshift**.
Zero-pad = интерполяция углового спектра: разрешение определяет **реальная** апертура (i, j), но
число бинов после FFT = `N_pad` (больше). Реализовано в коде (`core/models/angular_fft.py`,
`fft3d.py`, `waveform_to_cube.py`), проверено на неквадрате 6×15 → padded 8×16.
