## 1. Проект
Способ, устройство и статья: распознавание истинной цели на фоне **самоприкрывающих** помех (DRFM/ISRJ, SMSP, заградка, RFI) для моностатического UHF-радара (0.5 ГГц) с АФАР. Прототип RU2549375C1 (два канала + корреляция, без структурного различителя цель/ложь). Приоритеты: надёжность → скорость → красота (единство).
Железо: **RX 9070** (RDNA4/gfx1201, WMMA 16×16×16, wave32, ~640 ГБ/с) и **MI100** (CDNA1, MFMA нативно 16×16 и 32×32, wave64, 1.23 ТБ/с). ROCm/HIP, rocFFT/hipFFT. Коррелятор FM-m в `AlexLan73/GPUWorkLib`.

## 2. СТЕРЖЕНЬ (главное!)
**Одно ядро** (объёмный токенизатор → ИИ-гейт → причинностный арбитр → агильный опрос FM-m) со **сменными интерфейсами** под тип сигнала. Токен на объёме (куб `16×16×L`) — **универсальный интерфейс**; новый сигнал = новый фронтенд, ядро не трогаем. Плоскость = частный случай `L=1`. N переменна.
```
сигнал → ИНТЕРФЕЙС (ЛЧМ: два FFT, точно | AM: скользящий 3D-FFT, грубо/быстро)
       → общий куб 16×16×L → ЯДРО (токенизатор→гейт→арбитр) → опрос FM-m → целеуказание
```

## 3. Такт (когнитивная петля)
ЛЧМ/AM заполняет куб → токены → гейт + арбитр (передний край τ≥0 / свежесть кода) → многолучевой опрос FM-m в выбранные площади → пауза → соседние. Дешёвый инвариант гейтит дорогой агильный зонд; физика — арбитр, нейросеть — адъюнкт.

## 4. Интерфейсы, заполняющие куб
- **ЛЧМ (точность):** глобальный дальностный FFT + поячеечный угловой 16×16 (это НЕ 3D-FFT — тон на всю запись). Δr=c/(2ΔF) (девиация, не база); sinθ=k/8. Данные по-антенно → дальностный FFT без копии; corner-turn перед угловым — слить в Kernel B (LDS).
- **AM (присутствие):** локальный **3D-FFT** по скользящему кубу с переменным шагом (8/16/32/64), произвольный под-куб в любой точке. Третья ось = дальность (грубо = позиция окна, тонко = kz). 3D нужен: локальный отклик → колокол, непрерывный (RFI) → размазан/провал. «Летит» — трекингом МЕЖДУ тактами, не из куба.

## 5. Токенизатор (одно ядро, 2D и 3D)
Признаки страддл-инвариантные: PR=(Σp)²/Σp², Hoyer, доля лепестка (2D 3×3 / 3D 3×3×3), интегральное отношение (охрана 2D 5×5 / 3D 5×5×5), max/mean, энергия. Порог — OS-CFAR (в 3D по объёму). Токен: пики (kx,ky[,kz],ярк,кромка)+признаки+метка. Гейт MLP 6→16→3 — гейт, НЕ арбитр. Всё в LDS, угловой FFT на тензорных блоках.

## 6. Арбитр (сердце) + L3
Причинность **τ≥0** (репитер только задерживает → ложные дальше). Два варианта: **передний край** (геометрия, на ЛЧМ/кубе) + **свежесть кода** (на опросе FM-m: враг не ответит на код, что не слышал). Область — самозащитная помеха. Объект↔RFI: объект локализован (один блок), RFI — во всех блоках дальности (полоса) + застывший тон → отсекается тем же алгоритмом + код-арбитром.
**L3** — LSTM/GRU над **токенами** ROI (не сырьё): вход — поток токенов + позиция/Δr; выход — только уточнённые (угол, дальность); конечная точка — **код-корреляция FM-m** (сеть наводит, физика решает; точность из интерполяции пика). Обучение на токенах (генератор даёт полный куб, но учим на токенах) — потом.

## 7. FM-m коррелятор
Многолучевой опрос по площади (S лучей × K гипотез задержки текущего кода, параллельно на GPU). K опор = дальностные стробы текущего кода (НЕ история). Кернел токенизации на выходе IFFT (3–5 максимумов). Длина кода агильная 2⁸…2²⁰, меняется каждый такт; эталонного полинома нет. Коллапс S×K→S — опция (опоры суть сдвиги одного кода).

## 8. Формула
Новизна — СОВОКУПНОСТЬ 4 элементов: (а) агильная длина 2ⁿ + два FFT; (б) страддл-инвариантный токен; (в) ИИ-гейт + причинностный арбитр; (г) целеуказание агильного FM-m из матрицы токенов. + объёмный примитив (токен 16×16×L, инвариантность к фронтенду) — зависимые.
Способ и устройство — **раздельные заявки** (устройство: структурная схема с позициями; способ: схема осуществления).

## 10. Бюджет времени (L1 без FM-m)
Кадр 16×16×N: 0.5М = 1.0 ГБ, 1М = 2.0 ГБ. MI100 (1.23 ТБ/с): грубый скан шаг 64 ≈ 0.2–0.3 мс (0.5М) / 0.4–0.5 мс (1М); сплошной шаг 16 ≈ 1/2 мс; под-куб (зум) — микросекунды. Слитый Kernel B держит субмиллисекунду; раздельные ядра почти вдвое дороже.
