# Глава 2. Грубое обнаружение на угловой карте: признаки, триаж и сборка структуры

## 2.0. Место главы в пайплайне

К этому моменту (Глава 1) сырые данные после гетеродина `16×16×N_r`
(две оси апертуры и ось быстрого времени) уже прошли **дальностный БПФ** по
третьей оси. Поэтому на вход настоящей главы поступает комплексный тензор

```
S[k_x, k_y, r] ,   размер 16 × 16 × N_r ,   N_r = 10000
```

где `r` — номер бина дальности (ось Z теперь означает дальность, а не время).
Для каждого фиксированного `r` срез `S[:, :, r]` — это комплексная
**апертурная матрица 16×16** (сигнал на решётке антенных элементов для данной
дальности).

Задача главы: по каждому такому срезу быстро решить, есть ли в нём собранный
источник и какого он типа, и упаковать результат в компактный «токен». Всё это
выполняет один вычислительный блок — **ядро B**.

---

## 2.1. Обозначения

| Символ | Смысл | Размер / диапазон |
|---|---|---|
| `N` | сторона апертуры | 16 |
| `n = N·N` | число угловых ячеек | 256 |
| `r` | индекс бина дальности | 0 … N_r−1 |
| `S[k_x,k_y,r]` | комплексный сигнал на элементах решётки | 16×16, ℂ |
| `A[k_x,k_y]` | комплексная **угловая карта** после углового БПФ | 16×16, ℂ |
| `P[k_x,k_y] = |A|²` | **энергетическая угловая карта** | 16×16, ℝ≥0 |
| `p_i` | значения `P`, развёрнутые в вектор длины `n` | i = 0 … 255 |
| `(k_x, k_y)` | номер углового бина; связь с углом `sinθ = k/8` при d=λ/2 | −8 … 7 |

«Угловая карта» — это результат формирования луча по всем направлениям сразу:
каждая ячейка `(k_x, k_y)` соответствует своему направлению прихода, а её
значение — энергия, пришедшая с этого направления на данной дальности.

---

## 2.2. Что делает ядро B (обзор)

Ядро B обрабатывает **одну угловую матрицу** `S[:, :, r]` и выдаёт один токен.
Пять шагов:

```
вход:  S[:, :, r]                        — апертурная матрица 16×16 (комплекс)
  1. A = угловой БПФ 2D (S · окно)       — формирование луча, остаётся комплексом
  2. P = |A|²                            — энергетическая карта (переход в float)
  3. признаки f = features(P)            — вектор инженерных признаков
  4. пики = поиск пиков по углу (до 5)   — координаты и яркости источников
  5. (метка, скор) = MLP(f)              — классификация типа среза
выход: токен(r) = { пики[], признаки, метка, скор }
```

Важное свойство порядка: **магнитуда `|A|²` берётся только после углового БПФ.**
Угловой БПФ — это когерентное суммирование по апертуре, ему нужна фаза между
элементами; если взять модуль раньше, направление восстановить нельзя.

> Тайлинг под GPU (как несколько срезов группируются на один вычислительный
> блок ради занятости) описан отдельно в разделе 2.9 и на логику не влияет.

---

## 2.3. Шаг 1 — угловой БПФ (формирование луча)

Применяем двумерное БПФ к апертурной матрице, предварительно домножив на
оконную функцию `w` (Хэмминга) по обеим осям для подавления боковых лепестков:

```
A[k_x,k_y] = FFT2( S[:,:,r] · w(x)·w(y) )
```

Результат `A` — комплексная угловая карта 16×16. Точечный источник с одного
направления даёт узкий главный лепесток в одной ячейке; шум — равномерную рябь.

---

## 2.4. Шаг 2 — энергетическая карта

```
P[k_x,k_y] = |A[k_x,k_y]|²
```

С этого шага работаем с вещественной картой `P` (энергия по направлениям).
Дальше все признаки считаются по `P`.

---

## 2.5. Шаг 3 — линейка признаков

Классифицировать срез по всем 256 значениям карты дорого (тяжёлая сеть).
Вместо этого сжимаем карту в короткий **вектор инженерных признаков** — каждый
описывает одно свойство карты и считается дешёвой редукцией. Обозначим
`p_i = |A_i|²`, `Σp = Σ p_i`, `Σp² = Σ p_i²`.

### 2.5.1. Признаки собранности (устойчивые якоря)

**Participation Ratio (PR) — эффективное число «горящих» ячеек:**

```
PR = (Σp)² / Σp²
```

Смысл: если вся энергия в одной ячейке — `PR ≈ 1`; если размазана по всем 256
поровну — `PR ≈ n`. Для шума `PR ≈ n/2 = 128` (следствие статистики шума).
Малый PR = собранный источник.

**Индекс Хойера (Hoyer) — нормированная разреженность в [0,1]:**

```
Hoyer = (√n − ℓ1/ℓ2) / (√n − 1) ,   ℓ1 = Σp ,   ℓ2 = √(Σp²)
```

`Hoyer → 1` — предельно собрано, `→ 0` — размазано.

### 2.5.2. Признаки формы главного лепестка (страддл-устойчивые)

Если источник попадает между угловыми бинами (страддл), его энергия делится
между соседними ячейками. Чтобы это не сбивало классификацию, меряем не
отдельную ячейку, а **лепесток целиком** — блок 3×3 вокруг максимума.
Пусть `(i,j) = argmax P`, `Block(i,j)` — сумма по окну 3×3 вокруг него.

**Доля главного лепестка:**

```
MainFrac = Block(i,j) / Σp
```

Доля энергии, собранная в главном лепестке. Точечная цель (даже при страддле)
→ ~0.94–0.98; заградка → ~0.40; шум → ~0.07.

**Интегральное отношение лепестков** (устойчивая замена «2-й/1-й пик»):
обнуляем охранную зону 5×5 вокруг главного пика, находим второй по силе
лепесток `Block2`, берём отношение:

```
LobeRatio = Block2 / Block(i,j)
```

Один источник → `LobeRatio ≈ 0` (второго лепестка нет); много источников /
заградка → заметно больше нуля; шум → ≈ 1.

### 2.5.3. Вспомогательные признаки

```
MaxMean  = max(P) / mean(P)     — контраст пика (яркость)
PeakPos  = argmax P → (k_x,k_y) — направление главного источника
Energy   = Σp                   — абсолютный уровень
```

### 2.5.4. Итоговый вектор

```
f = [ PR, Hoyer, MainFrac, LobeRatio, MaxMean, Energy ]   (+ PeakPos отдельно)
```

Все компоненты — 1–2 прохода по 256 значениям; в вычислительном ядре это
редукции в быстрой памяти. Набор не финальный: по результатам обучения лишние
убираются, при необходимости добавляются (энтропия карты и т. п.).

---

## 2.6. Шаг 4 — поиск пиков по углу (массив источников)

На одной дальности может быть несколько источников под **разными углами**
(разные направления, одна дальность). Поэтому в токен пишем не одну точку, а
**массив пиков фиксированной длины** (до 5, с запасом):

```
пики[5] = { (k_x, k_y, яркость, кромка) }
n_пиков  = сколько из 5 валидны (0…5)
```

- `кромка` — нарастание/спад пика по соседним бинам дальности; почти бесплатна
  (соседние бины уже под рукой) и служит физическим признаком (резкая кромка
  реального отражателя против размытой копии);
- если пиков больше 5 — это само по себе признак «размазано» (заградка),
  срез метится соответственно, лишние пики не пишутся.

Важно: массив ловит множественность **по углу**. Множественность **по
дальности** (гребёнка повторителя) здесь не решается — её собирает Проход 2
(раздел 2.8). Две оси структуры независимы: угол — массивом, дальность —
сборкой.

---

## 2.7. Шаг 5 — триаж (классификатор типа среза)

Вектор признаков подаётся на компактную нейросеть (MLP):

```
MLP: 6 входов → 16 нейронов → 3 класса
классы среза:  шум  /  собранный источник  /  размазанный (заградка)
выход: метка + скор (уверенность)
```

Только **три** класса, потому что по одной угловой карте физически различимы
только они. Гребёнка повторителя от одиночной цели по одному срезу
**неотличима** (они различаются по дальности) — её выделяет Проход 2.

**Триаж — приоритизатор, не арбитр.** Метку «цель» окончательно ставит физика
на следующем уровне (передний край + корреляция с текущим кодом), а не эта
сеть: повторитель даёт «собранный источник», для угловой сети неотличимый от
настоящей цели. Поэтому выход сети — это скор для приоритета обработки, а не
окончательный вердикт. Порог записи токена делается мягким, чтобы слабый
ближний пик (истинная цель под яркой ложной) не отсеялся.

---

## 2.8. Проход 2 — сборка структуры по дальности

Токены проходятся во второй, дешёвый проход. Берём токены с меткой «собранный
источник» под одним направлением `(k_x, k_y)` и раскладываем по дальности `r`:

| Картина токенов вдоль дальности | Вывод |
|---|---|
| одиночный токен | одиночная цель |
| несколько с равным шагом Δr | гребёнка повторителя; Δr = задержка петли |
| **ближний токен группы** | **передний край = истинная цель** |
| токены во всех r подряд | заградка / непрерывная помеха |

- **Передний край**: истинная цель — ближний пик группы, а не самый яркий.
  Повторитель может только задержать сигнал, поэтому ложные цели всегда дальше
  истинной. Это физический признак, который нельзя подделать.
- **Период** гребёнки находится автокорреляцией цепочки токенов; шаг `Δr`
  переводится в задержку петли повторителя.

Обработка идёт по единицам токенов (не по кубу данных) → очень дёшево. Её
результат — целеуказание для активного зондирования.

**Итог по классам:** четыре типа сцены живут на двух уровнях —
на срезе (Проход 1): шум / собранный / размазанный; на профиле (Проход 2):
«собранный» распадается на одиночную цель и гребёнку повторителя.

---

## 2.9. Реализация на GPU (оптимизация)

Логика выше не зависит от того, как срезы раскладываются по вычислительным
блокам. Ради занятости GPU один блок берёт **плитку из 16 соседних бинов
дальности**: грузит `S[:, :, r … r+15]` в быструю память, выполняет угловой
БПФ и все признаки для каждой из 16 матриц, наружу пишет только токены.
Тяжёлый комплексный куб не выгружается в общую память между шагами — угловой
БПФ, магнитуда, признаки и триаж слиты в один проход по быстрой памяти. Это и
есть источник скорости; на смысл алгоритма плитка не влияет.

## 2.10. Проверка на модельной сцене

Признаки посчитаны на модельных угловых картах 16×16 (окно Хэмминга, комплексный
гауссов шум). Значения — иллюстративные, зависят от отношения сигнал/шум и
параметров сцены; важна **структура разделения**, а не конкретные числа.

**Три класса (источник на сетке):**

| класс | PR | Hoyer | MaxMean | MainFrac | LobeRatio |
|---|---|---|---|---|---|
| шум | 129 | 0.31 | 5.4 | 0.07 | 1.03 |
| цель | 3.6 | 0.94 | 123 | 0.98 | 0.002 |
| заградка | 15–23 | 0.81 | 22–42 | 0.40 | 0.25 |

**Устойчивость к страддлу (источник смещается на полбина):**

| признак | цель на сетке | цель на полбина | вывод |
|---|---|---|---|
| PR | 3.6 | 4.8 | устойчив |
| Hoyer | 0.94 | 0.92 | устойчив |
| MainFrac | 0.98 | 0.94 | устойчив |
| LobeRatio | 0.002 | 0.002 | устойчив |
| «2-й/1-й пик» (сырой) | 0.21 | 0.99 | ломается → не используется |

Разделение классов держится на устойчивых якорях (PR, Hoyer, MainFrac,
LobeRatio); сырой пиковый признак чувствителен к страддлу и оставлен только
как исторический.

(устойчивость к смещению).

## 2.11. Открытые вопросы

1. Итоговый состав вектора признаков — уточняется по результатам обучения.
2. Пороги классов и порог записи токена — калибруются на синтетическом датасете
   по заданной вероятности ложной тревоги.
3. Нерегулярная структура по дальности (рваная гребёнка, дрожащая задержка) —
   выносится на глубокий уровень (патч по дальности + объёмная сеть), глубина
   патча (64/128/256) определяется типичным периодом гребёнки — тестируется.
