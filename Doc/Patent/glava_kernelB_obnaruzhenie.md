# Глава. Обнаружение на угловой карте: признаки, триаж, сборка по дальности

## Кернел B радарного ECCM-каскада

---

## 1. Введение и место главы в конвейере

Данные после гетеродина — комплексный тензор `16×16×N`, где `16×16` — приёмная
апертура (азимут × угол места), а `N` — отсчёты быстрого времени (дальность).
Обработка идёт в два кернела:

- **Кернел A** — дальностный БПФ по оси быстрого времени. Переводит дечирпованный
  сигнал в дальность: на выходе тот же тензор `16×16×N`, но теперь третья ось —
  готовые бины дальности. Данные остаются **комплексными** (фаза ещё нужна).
- **Кернел B** — предмет этой главы. Для каждого бина дальности берёт его угловую
  матрицу `16×16`, формирует луч (угловой БПФ), считает признаки, классифицирует
  и выдаёт компактный токен.

Настоящая глава описывает Кернел B: как из одной угловой матрицы получается
решение «шум / источник / помеха» и как последовательность таких решений вдоль
дальности восстанавливает структуру цели и отделяет её от помехи ретранслятора.

---

## 2. Обозначения и переменные

| Обозначение | Смысл |
|---|---|
| `N` | число бинов дальности (отсчётов быстрого времени) |
| `n` | индекс бина дальности, `n = 0 … N−1` |
| `A[kx, ky]` | комплексная угловая матрица `16×16` одного бина дальности |
| `kx, ky` | индексы угловых бинов (азимут, угол места), `−8 … 7` после `fftshift` |
| `P[kx, ky]` | карта энергии, `P = |A|²` (вещественная, `16×16`) |
| `M = 256` | число ячеек угловой карты (`16×16`) |
| `p_i` | энергия `i`-й ячейки, `i = 0 … 255` (развёрнутая `P`) |
| `S1` | сумма энергии, `S1 = Σ p_i` |
| `S2` | сумма квадратов, `S2 = Σ p_i²` |
| `токен` | компактная запись найденного в одном бине дальности |

Все признаки считаются по одной угловой карте `P`. «Источник» — любой собранный
по углу сигнал (цель или ретранслятор); «помеха» здесь — заградительная (шумовая
из сектора углов).

---

## 3. Метод Кернела B: последовательность операций

Базовая единица обработки — **одна угловая матрица `16×16`** (один бин дальности).
Для неё выполняется:

1. **Угловой БПФ (формирование луча).** Двумерный БПФ по апертуре переводит
   матрицу элементов в матрицу направлений. Операция когерентная — складывает
   комплексные вклады 256 элементов, поэтому вход обязан быть комплексным.
   Реализуется как матричное умножение на тензорных ядрах (WMMA).

2. **Переход к энергии.** `P = |A|²`. Здесь данные становятся вещественными
   (`float`). Дальше вся работа — по `P`.

3. **Расчёт признаков.** По карте `P` считается вектор чисел (раздел 4). Это
   дешёвые редукции (суммы, максимумы) по 256 значениям.

4. **Поиск пиков по углу.** В карте выделяется до 5 локальных максимумов — это
   разные источники под разными направлениями на одной дальности (раздел 5).

5. **Классификация (триаж).** Малая нейросеть по вектору признаков ставит метку
   среза: `шум / источник / заградка` и скор уверенности (раздел 6).

6. **Формирование токена.** Если срез не «шум» — записывается токен со структурой
   (раздел 5). Пустые срезы не пишутся → выход разрежённый.

Решение о наличии цели на этом уровне **не** выносится: см. раздел 8 (роль триажа)
и раздел 7 (сборка по дальности).

## 4. Признаки угловой карты

Признаки описывают, **насколько энергия собрана** в карте `P`. Настоящая цель
собирает энергию в один лепесток; шум размазан по всем ячейкам; заградка
размазана по сектору. Именно эту «собранность» и меряют признаки.

### 4.1 Participation Ratio (PR) — эффективное число зажжённых ячеек

```
PR = (Σ p_i)² / Σ p_i² = S1² / S2
```

Смысл: сколько ячеек реально несут энергию. Точечный источник → `PR ≈ 1–3`;
заградка → десятки; шум → `≈ M/2 = 128` (значение для экспоненциально
распределённого шума). Главный якорь «источник против шума».

### 4.2 Hoyer — мера разреженности, [0, 1]

```
Hoyer = (√M − S1/√S2) / (√M − 1)
```

`Hoyer → 1` для очень собранной карты, `→ 0` для равномерной. Второй устойчивый
якорь собранности.

### 4.3 Доля главного лепестка — устойчивая к смещению цели

Пусть `(i*, j*) = argmax P`. Главный лепесток — блок `3×3` вокруг максимума:

```
Lmain = Σ P[i, j],  (i, j) ∈ блок 3×3 вокруг (i*, j*)
main_frac = Lmain / S1
```

Смысл: какая доля всей энергии сидит в главном лепестке. Цель → 0.94–0.98
(даже если пик расщеплён между соседними ячейками при смещении с сетки);
заградка → ~0.4; шум → ~0.07. Устойчива к straddle (цель между бинами).

### 4.4 Интегральное отношение второго лепестка к первому

Обнуляем охранную зону `5×5` вокруг главного пика, ищем второй лепесток:

```
Lsecond = сумма блока 3×3 вокруг максимума ВНЕ охранной зоны
lobe_ratio = Lsecond / Lmain
```

Смысл: есть ли в карте второй сравнимый источник. Одиночная цель → ~0.002
(второго лепестка нет); заградка → ~0.25 (много сравнимых); шум → ~1.0.
Устойчивая к смещению замена «сырому» отношению двух пиков.

### 4.5 Вспомогательные признаки

```
max_mean = max(p_i) / mean(p_i)      — яркость пика (слабо плывёт при смещении)
peak_pos = (kx*, ky*)                — направление главного пика
energy   = S1                        — абсолютный уровень энергии
```

### 4.6 Сводка и приоритет

| Признак | Что меряет | Устойчивость к смещению |
|---|---|---|
| PR | собранность (источник/шум) | устойчив |
| Hoyer | разреженность | устойчив |
| main_frac | доля главного лепестка | устойчив |
| lobe_ratio | одиночный/множественный | устойчив |
| max_mean | яркость | слабо плывёт |
| peak_pos | направление | — |
| energy | уровень | — |

Опора решения — четыре устойчивых признака (PR, Hoyer, main_frac, lobe_ratio).
Ни один признак поодиночке не разделяет все классы; разделяет их вектор
(проверка на числах — раздел 9).

## 5. Токен: результат обработки одного бина дальности

На одной дальности может быть несколько источников под разными углами, поэтому
токен хранит **массив пиков** фиксированной длины (на GPU нет динамической памяти):

```
токен = {
    n,                         // индекс бина дальности
    n_peaks,                   // сколько пиков реально найдено (0..5)
    peaks[5] = {               // массив фиксированной длины 5 (с запасом)
        kx, ky,                // направление пика
        amp,                   // амплитуда (яркость)
        edge                   // передний фронт по дальности (см. ниже)
    },
    PR, Hoyer, main_frac, lobe_ratio,   // признаки среза
    label,                     // метка триажа: шум / источник / заградка
    score                      // уверенность метки
}
```

- `n_peaks` — сколько из пяти ячеек массива валидны; если пиков больше пяти
  (сильная заградка), это само по себе признак «размазано», лишние не пишутся;
- `edge` — оценка переднего фронта пика по соседним бинам дальности (нарастание/
  спад). Даёт частичный передний край уже здесь и отличает резкую кромку реального
  отражателя от размытой копии.

Две оси структуры разведены: **по углу** источники ловит массив `peaks`;
**по дальности** структуру (гребёнку) собирает раздел 7.

---

## 6. Триаж: классификация среза

Классификатор — малая нейросеть: вход = вектор признаков (раздел 4), выход = три
класса и скор:

```
вход: [PR, Hoyer, main_frac, lobe_ratio, max_mean, energy]  (6 чисел)
сеть: 6 → 16 → 3
выход: label ∈ {шум, источник, заградка}, score ∈ [0,1]
```

Веса сети — единицы килобайт, помещаются в быстрой памяти рядом с картой.

Три класса — это то, что физически различимо по **одной** угловой карте.
Четвёртый случай, гребёнка ретранслятора, по одной карте неотличим от цели
(отличие — по дальности) и определяется на сборке (раздел 7).

Порог записи токена делается **мягким**: слабый ближний пик (истинная цель под
яркой ложной) не должен отсеяться — передний край терять нельзя.

---

## 7. Сборка по дальности: гребёнка и передний край

Второй проход работает уже не с картой, а с потоком токенов. Собираем токены с
меткой «источник» под одним направлением `(kx, ky)` и раскладываем по `n`:

| Картина токенов вдоль дальности | Вывод |
|---|---|
| один токен | одиночная цель |
| несколько с равным шагом `Δn` | гребёнка ретранслятора; `Δn` = задержка петли |
| ближний токен группы (`min n`) | передний край = истинная цель |
| токены во всех `n` подряд | заградка / сплошной источник |

Физический принцип: ретранслятор способен только **задержать** сигнал, поэтому
ложные отметки всегда **дальше** истинной. Истинная цель — на **переднем крае**
(минимальная дальность в группе), а **не** самая яркая. Период `Δn` находится
автокорреляцией цепочки токенов; передний край — минимальным `n`.

Итог по классам разнесён по уровням:

- срез (проход 1): `шум / источник / заградка` — различимо по углу;
- профиль (проход 2): `источник` распадается на `цель / гребёнку` — различимо по
  дальности (передний край + период).

---

## 8. Роль триажа: приоритизатор, не арбитр

Нейросеть триажа **упорядочивает** кандидатов, но не выносит окончательный вердикт
«цель». Причина: ретранслятор даёт «источник», по углу неотличимый от настоящей
цели. Поэтому:

- выход сети (`label`, `score`) задаёт приоритет обработки;
- окончательную метку «цель / ложная» ставит физика: передний край (раздел 7) и
  корреляция с текущим кодом зондирования на следующем уровне каскада.

Это требование устойчивости к адаптивному противнику: подтолкнуть скор сети можно,
причинность переднего края — нельзя.

## 9. Проверка на числах (модельная сцена)

Признаки посчитаны на модельных угловых картах `16×16` (аподизация Хэмминга,
комплексный гауссов шум).

### 9.1 Три класса, цель на сетке

| класс | PR | Hoyer | max_mean | lobe_ratio | main_frac | energy |
|---|---|---|---|---|---|---|
| шум | 129 | 0.31 | 5.4 | 1.03 | 0.07 | 15 649 |
| цель | 3.6 | 0.94 | 123 | 0.002 | 0.98 | 596 120 |
| заградка | 23 | 0.81 | 22 | 0.25 | 0.40 | 362 432 |

PR отделяет шум (≈128) от источника (<25); `lobe_ratio` и `main_frac` отделяют
цель от заградки. Ни один признак в одиночку не разделяет все три — разделяет
вектор.

### 9.2 Устойчивость к смещению цели (straddle)

Цель сдвигается с сетки от 0 до 0.5 бина:

| смещение | PR | main_frac | lobe_ratio (интегр.) | 2/1 пик (сырой) |
|---|---|---|---|---|
| 0.00 | 3.6 | 0.98 | 0.002 | 0.21 |
| 0.25 | 4.3 | 0.97 | 0.002 | 0.50 |
| 0.50 | 4.9 | 0.94 | 0.002 | 0.99 ⚠ |

«Сырое» отношение двух пиков при смещении улетает к 0.99 (путается с заградкой);
интегральные `main_frac` и `lobe_ratio` держатся → устойчивы. Это обоснование
выбора признаков из раздела 4.

### 9.3 Сборка по дальности

Токены под одним углом `(n, amp)`: `(60, 1.0), (78, 2.5), (96, 2.5), (114, 2.5)`.

- передний край `min n = 60` → истинная цель (хотя она слабее);
- период `Δn = 18` (автокорреляция) → задержка петли ретранслятора;
- ярчайший токен `n = 78` — ложная (дальше края).

Обычный детектор выбрал бы ярчайший (ложную); правило переднего края берёт `n=60`.

### 9.4 Оговорка

Числа зависят от модели сцены (SNR, ширина сектора заградки, аподизация). Пороги —
стартовые; финальная калибровка на синтетическом наборе по заданной вероятности
ложной тревоги. Устойчива структура разделения, а не конкретные значения.

## 10. GPU-оптимизация (тайлинг)

Базовая единица описана как одна угловая матрица. На GPU для эффективности один
workgroup обрабатывает **плитку из нескольких соседних бинов дальности** (например,
16), загружая их в быструю память (LDS) одним блоком: угловой БПФ и признаки
считаются для каждой матрицы плитки, наружу пишутся только токены. Это деталь
реализации; логика обработки одной матрицы (разделы 3–6) от неё не меняется.

## 11. Открытые вопросы

1. Состав признаков: стартовый набор из раздела 4; уточняется по результатам
   обучения (кандидат на добавление — энтропия карты).
2. Порог триажа: по скору сети или отдельный ранний порог по PR (быстрый отсев
   явного шума до нейросети).
3. Глубина патча для анализа нерегулярной структуры (рваная гребёнка) на более
   высоком уровне каскада: 64 / 128 / 256 бинов — определяется по типичному
   периоду гребёнки, требует тестирования.
