# План документа: способ и устройство распознавания цели на фоне самоприкрывающих помех

## Скелет статьи и патента · тезисы по главам

**Назначение:** единый источник, из которого готовятся (1) статья и (2) заявка на патент (способ + устройство).
**Стиль:** технический, для инженеров; формулы там, где несут смысл.
**Приоритеты:** надёжность распознавания → скорость обработки → красота решения.

Условные пометки: `[Ф]` — здесь формула · `[Рис]` — здесь рисунок · `[Хвост]` — недоделанный кусок · `[?]` — открытый вопрос к согласованию.

**Базовые параметры:**
- Апертура каскада — **плитка 16×16** (256 угловых ячеек); `n=256` означает 16×16 ячеек угловой карты.
- Глубина дальности **N — переменная** за такт. Тензор — **16×16×N**. Размер дальностного FFT ставится под текущее N (дополнение нулями до 2ⁿ).
- Длина кода FM-m **агильная, 2⁸…2²⁰**, меняется каждый такт.
- Термины — см. **Приложение Б (Глоссарий)**.

## Такт работы

Двухэтапная когнитивная петля. Один такт:

1. **ЛЧМ широко** освещает сектор — грубо определяем, «куда смотреть».
2. Приняли антеннами, посчитали объём **16×16×N** (два раздельных FFT), свернули каждый срез дальности в **токен**, лёгкий ИИ-**гейт** + физический **арбитр переднего края** → нашли точки/цель и координаты.
3. В нужный квадрат послали **FM-m** (агильный код) — точечный опрос/подтверждение через коррелятор.
4. **Пауза**; радар может пройтись по соседним квадратам. Всё на GPU стёрли, новый такт — новый код, новая длина, новое N.

Ключ решения: **дешёвый инвариант (ЛЧМ-картинка → токены) гейтит дорогой агильный зонд (FM-m)**; физика — арбитр, нейросеть — адъюнкт.

## Часть I. Постановка

### Глава 1. Задача и модель угрозы
**Тезисы:**
- Моностатический UHF-радар (0.5 ГГц) с АФАР; два зонда — широкий ЛЧМ (поиск) и точечный фазовый M-код FM-m (опрос).
- Угроза — **самоприкрытие**: DRFM/ISRJ (гребёнка ложных целей, ярче истинной), SMSP (размытие спектра), заградка, сторонний непрерывный источник (RFI).
- Прототип RU2549375C1 не тянет самоприкрытие: два канала (основной+вспомогательный) и корреляция для компенсации помехи в главном луче; отдельного структурного различителя цель/ложь нет. `[Ф: почему прототип ломается на самоприкрытии]`
- Наш ответ — двухэтапная петля (см. «Такт работы») с причинностным арбитром.

### Глава 2. Тракт приёма и первичная обработка
**Тезисы:**
- Аппаратная цепочка: АФАР → АЦП → **гетеродин-дечирп на ПЛИС** (stretch-processing) → сырые IQ.
- Дечирп превращает задержку цели в постоянную частоту биений. `[Ф: f_b = μ·2R/c, μ = ΔF/T_c]`
- После дечирпа поток за такт — **N комплексных отсчётов** (до ~1.3М на канал), N переменная.
- Батч-FFT по всем антеннам на GPU за один вызов; размер плана — под текущее N (до 2ⁿ).
- **После дечирпа частота = дальность** (одна цель → один пик; гребёнка → ряд пиков с шагом Δf). `[Рис: спектр чистой цели vs гребёнка]`

## Часть II. Каскад: объёмный анализ и токены

### Глава 3. Геометрия данных и два FFT
**Тезисы:**
- Вход в каскад: тензор **16×16×N** (апертура×апертура×дальность), N переменная.
- **Два раздельных FFT в разных местах конвейера:**
  1. **Дальностный FFT** — глобальный, один раз, по оси быстрого времени (rocFFT). После него Z = дальность.
  2. **Угловой FFT 16×16** — поячеечно, на каждом бине дальности (WMMA/тензорные ядра).
- Магнитуду `|·|²` берём **только после углового FFT** (он когерентно складывает по апертуре — нужна фаза).
- Метрика **параметрическая** (меняется на ходу, алгоритм постоянен). `[Ф: Δr = c/(2ΔF); sinθ = k·λ/(N_ап·d) = k/8 при d=λ/2, k=−8…7]`.
- `[Хвост]` таблица рабочих параметров: девиация ΔF, частота дискретизации f_s, шаг решётки d — зафиксировать диапазоны.

### Глава 4. L1 — грубое обнаружение на угловой карте (токенизация)
**Тезисы:**
- Угловая карта `P = |A|²` (256 = 16×16 ячеек); классификация **не по сырым 256 значениям, а по вектору инженерных признаков**.
- Признаки: **PR** `=(Σp)²/Σp²` `[Ф]`, **Hoyer** `[Ф]`, **доля главного лепестка (3×3)**, **интегральное отношение лепестков** `[Ф]`, max/mean, энергия; `p_i=|A_i|²`, n=256. Страддл-устойчивые якоря — PR, Hoyer, доля лепестка, интегральное отношение.
- Проверено: цель на сетке и в страддле по якорям неотличимы, а от заградки отделены чисто. `[Рис] feat_scene, straddle, intfeat`.
- **Токен со структурой**: массив до 5 пиков `(kx,ky,яркость,кромка)` + признаки + метка. Две ортогональные оси: **массив пиков — по углу**, **сборка токенов — по дальности**. `[Рис] tokenize`.
- **Триаж MLP** 6→16→3 (шум / собранный / размазанный) — это **ГЕЙТ, не арбитр**: DRFM даёт «собранный», для угловой сети неотличимый от цели; метку цель/ложь ставит физика (гл. 5).
- **Проход 2** (сборка вдоль дальности): передний край (ближний ≠ яркий) + период гребёнки (автокорреляция цепочки токенов). `[Ф: Δr_гребёнки]`
- `[Хвост]` проход 2 на злых сценах: рваная гребёнка, две цели под разными углами на одной дальности.

### Глава 5. L2 — физический арбитр: правило переднего края (сердце заявки)
**Тезисы:**
- **Причинность τ ≥ 0.** DRFM/репитер обязан сначала принять, потом переизлучить → его ложная цель стоит на той же дальности или **дальше**, никогда не ближе. Ось односторонняя. `[Ф: τ ≥ 0]`
- **Правило переднего края (геометрия, на картинке ЛЧМ):** истинная цель — **ближний** пик причинностной группы; хвостовые пики с регулярным шагом (гребёнка) — кандидаты в ложные. Выбор **по дальности, не по яркости** («ближний ≠ яркий», DRFM бывает ярче).
- **Свежесть кода (сигнал, на этапе опроса FM-m):** код опроса **свежий и непредсказуемый** → враг не может подделать ответ на код, которого ещё не слышал. Его переизлучение либо несёт не тот код (в корреляторе не сжимается), либо приходит позже (дальше → снова передний край). Это свойство самого опроса FM-m.
- Два признака дополняют друг друга и работают **каждый на своём этапе**: геометрия ловит то, что упускает код, и наоборот. Помеха обязана обойти оба.
- Область действия правила — **самозащитная (самоприкрывающая) помеха** (на носителе цели); уводящий джаммер на *другом*, более близком носителе отделяется свежестью кода + разнесением по углу. Это ограничение — в область независимого пункта.
- `[Хвост]` сверить полином/длину LFSR в коде — длина агильная 2⁸…2²⁰, эталонного полинома нет; зафиксировать генератор для референс-реализации.

### Глава 6. FM-m коррелятор — реализация опроса на GPU
**Тезисы:**
- Опрос: заявка на код длины n и **k опор** → строим k сопряжённых опор (сдвиги/дальностные стробы текущего кода), берём FFT, держим на GPU. Приходит вход → FFT → перемножение с опорами → IFFT → первые j отсчётов → анализ пика → ответ → **стёрли, новый такт**.
- Корреляция через БПФ: `corr = IFFT(conj(FFT(ref))·FFT(inp))`, O(N log N). `[Ф]`
- Архитектура: **два HIP-потока** (эталон готовится один раз, переиспользуется; вход — параллельно), 4 слитых ядра (циклические сдвиги, сопряжённое умножение, извлечение магнитуд, генератор тест-входов). Микрооптимизации: aligned float2, побитовый модуль, FMUL вместо FDIV, эрмитова симметрия (half_N=N/2+1).
- **Батчинг по памяти (BatchManager):** влезает всё → один проход; не влезает → режет на батчи по ~70% свободной памяти, эталон не трогается, пересоздаются только буферы входа, данные подаются прямым указателем без копий.
- **Режим `S×K`:** коллапс `S×K → S` применим только когда опоры суть сдвиги одного кода (тест/калибровка). В работе опоры — по текущему коду такта, честные `S×K` IFFT.
- `[Хвост] Кернел токенизации на выходе IFFT:` сразу после IFFT встроить ядро, которое из корреляционного отклика достаёт **3/5 максимумов** и формирует набор токенов (позиция пика, амплитуда, кромка) вместо передачи полного отклика. Резко сокращает объём данных на дальнейшую обработку — та же логика «токен вместо сырья», что и в L1, но на выходе коррелятора. Согласуется с двумя ортогональными осями (пики — по стробу, сборка — по такту/дальности). `[?]` 3 или 5 значений — подобрать по сценам.
- `[Рис]` схема двухпоточного конвейера.

### Глава 7. L3 — глубокий добор
**Тезисы:**
- Патч по дальности + объёмная сеть **16×16×(64/128/256)** для **нерегулярной** структуры (рваная гребёнка, дрожащая задержка).
- Код-корреляция как основной инструмент; MUSIC/ESPRIT только для доброкачественного многоцелевого (под когерентной DRFM ломаются).
- Роль: добор на единицах сложных кандидатов, не основной поток.
- `[Хвост]` глубина патча — от типичного периода гребёнки (подтвердить тестами).

### Глава 8. Целеуказание FM+M пучка (замыкание петли)
**Тезисы:**
- Из L1/L2/L3: номер бина дальности, тонкая дальность, углы → пучок 8–10 лучей в конус неопределённости.
- Когнитивная петля (см. «Такт работы»): **ЛЧМ обнаружил (широко) → дал точки → FM-m зондировал (в квадрат) → пауза → соседние квадраты**.
- Целеуказание строится из **компактной матрицы токенов**, а не из сырого объёма — в этом скорость решения.

### Глава 9. Устойчивость к противнику и латентность
**Тезисы:**
- **Причинность как неподделываемый признак:** враг не ответит кодом, которого ещё не слышал (передний край + свежесть кода).
- Анти-затопление: потолок кандидатов K_max + приоритет по инвариантному скору → ограниченная латентность.
- Устойчивый CFAR (OS, локальный, по-угловой) против гребёнки/RFI.

**Тезисы:**

## Часть IV. Патентная формула (тезисы)

### Глава 12. Формула изобретения
**Ядро новизны — связка из 4 элементов:**
1. агильная **длина** кода 2ⁿ (n=8…20) с дополнением нулями под размер FFT;
2. редукция объёма 16×16×N в **страддл-инвариантный структурный токен** (массив пиков + признаки разреженности/лепестков);
3. **ИИ-гейт + физический арбитр** причинностного переднего края (τ≥0), а не end-to-end сеть;
4. **целеуказание агильного зондирующего пучка FM-m** из компактной матрицы токенов.

**Тезисы:**
- **Независимый п.1 (способ):** за такт — ЛЧМ-освещение; формирование 16×16×N двумя раздельными FFT; свёртка каждого среза в структурный токен из страддл-инвариантных признаков; лёгкий классификатор-гейт по классу среза, а метку цель/ложь ставит **правило ближнего края и/или согласования с текущим кодом** причинностной группы; из матрицы токенов — целеуказание следующего агильного пучка FM-m. Область: **самозащитная помеха**.
- **Зависимые:** конкретные признаки (PR/Hoyer/доля лепестка/интегральное отношение); объёмная сеть 16×16×(64/128/256) для нерегулярной структуры; двухпоточный GPU-коррелятор с батчингом по памяти; агильность длины 2⁸–2²⁰; CFAR OS по углу; энергетический триаж/ROI.
- **Независимый п.8 (устройство):** АФАР + ЦАП-формирователь агильного FM-m + GPU-конвейер двух FFT и токенизации + классификатор-гейт + блок арбитра переднего края. Зависимые: ЛЧМ-канал поиска, ROCm/hipFFT-коррелятор, блок целеуказания пучка.
- **Уровень техники (см. папку Files-Patet):** RU2549375C1 (прототип); US10962637B2 и US12181562 (NN на радарном кубе); US11747438 (когнитивный планировщик РЭБ); EP0097490A2 (FFT азимут/угол места по бинам); CN101441271A (GPU-SAR).
- **Патентная чистота** по классам (G01S 7/36, 7/292, 7/41, 7/38, 7/2813; H04K 3/22, 3/25) — обязательна, требует поверенного.

## Приложения

### Приложение А. Обозначения и параметры (сводная таблица) `[Хвост]`
Свести: ΔF (девиация), B (база), T_c, μ, f_s, d (шаг решётки), λ, N (глубина дальности), n (длина кода 2⁸–2²⁰), k (число опор/стробов такта), Δr, sinθ.

### Приложение Б. Глоссарий (обязательно)
- **АФАР** — активная фазированная антенная решётка.
- **UHF** — дециметровый диапазон; здесь 0.5 ГГц.
- **ЛЧМ** — линейно-частотно-модулированный сигнал (chirp); широкий зонд поиска.
- **FM-m / M-код** — фазоманипулированный код на основе m-последовательности; точечный зонд опроса.
- **LFSR** — регистр сдвига с линейной обратной связью, генератор m-последовательности.
- **m-последовательность** — псевдослучайная последовательность максимальной длины.
- **Дечирп / stretch** — смешение принятого ЛЧМ с опорным → задержка превращается в постоянную частоту биений.
- **Гетеродин** — узел смешения частот; здесь дечирп на ПЛИС.
- **IQ** — квадратурные (комплексные) отсчёты сигнала.
- **DRFM** — цифровая радиочастотная память; основа современных ретрансляционных помех.
- **ISRJ** — помеха с прерывистой выборкой и переизлучением (interrupted-sampling repeater jamming).
- **SMSP** — помеха размытия спектра (smeared spectrum).
- **Заградка** — заградительная (шумовая по полосе) помеха.
- **RFI** — сторонняя радиопомеха (напр. непрерывный источник, «радиолюбитель»).
- **Самоприкрытие** — помеха, излучаемая с носителя самой цели, прикрывающая её.
- **Гребёнка** — ряд ложных целей от DRFM с регулярным шагом по дальности.
- **FFT/БПФ, IFFT** — прямое/обратное быстрое преобразование Фурье.
- **rocFFT / hipFFT** — библиотеки FFT в стеке AMD ROCm.
- **ROCm / HIP** — платформа и язык GPU-вычислений AMD (аналог CUDA).
- **WMMA** — матрично-умножающие (тензорные) инструкции GPU; здесь угловой FFT 16×16.
- **RDNA4 / gfx1201** — архитектура/таргет GPU AMD RX 9070; wave32 — ширина волны 32.
- **VRAM / LDS** — видеопамять / локальная разделяемая память GPU.
- **Тензор 16×16×N** — объём апертура×апертура×дальность за такт.
- **Токен** — компактная запись среза: массив ≤5 пиков (kx,ky,яркость,кромка) + признаки + метка.
- **Страддл** — попадание пика между бинами (расщепление энергии на соседние ячейки).
- **PR (peak ratio)** — `(Σp)²/Σp²`, мера концентрации энергии.
- **Hoyer** — нормированная в [0,1] мера разреженности.
- **Главный лепесток (3×3)** — доля энергии в окне 3×3 вокруг максимума.
- **Интегральное отношение лепестков** — устойчивое отношение энергии главного к боковым.
- **CFAR (OS-CFAR)** — обнаружитель с постоянной вероятностью ложной тревоги (порядковой статистики).
- **MUSIC / ESPRIT** — сверхразрешающие методы оценки углов (ломаются под когерентной DRFM).
- **Beamspace** — обработка в пространстве лучей (не элементов).
- **Апертура** — раскрыв антенны; здесь плитка 16×16 элементов.
- **Δr = c/(2ΔF)** — разрешение по дальности (в знаменателе **девиация ΔF**, не база).
- **База B = T_c·ΔF** — произведение длительность×девиация; это помехозащищённость, не разрешение.
- **τ** — задержка (петли переизлучения); причинностно τ ≥ 0.
- **Передний край** — ближний по дальности член причинностной группы = истинная цель.
- **N** — глубина дальности (переменная, отсчётов после дечирпа).
- **k** — число сопряжённых опор/дальностных стробов на такт (текущего кода).
