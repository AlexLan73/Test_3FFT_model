# Шаг 1 — Геометрия данных и два FFT

## ЛЧМ-дечирп, один импульс, `16×16×10000` → угол × угол × дальность

**Дата:** 07.07.2026
**Платформа:** AMD RX 9070 (RDNA4/gfx1201, 16 GB VRAM) · ROCm 7.2 · rocFFT + WMMA

---

## 1. Что зафиксировано

1. Данные — ЛЧМ, дечирпованные. Дальность = частота биений.
2. Вход: ось Z (10000) — дечирпованное **время**, дальность в частоте, ещё не проявлена.
3. Первая операция — **глобальный дальностный FFT по всей оси Z** → Z становится дальностью (10000 бинов).
4. Затем — **поячеечный угловой FFT 16×16** на каждом бине дальности (WMMA).
5. Один импульс → доплера нет. Доплер добавляется позже 4-й осью (пачка импульсов).
6. Метрики (`Δr`, шкала, пороги) — параметрические, считаются из `ΔF, f_s`. Не прибиваем.

---

## 2. Два FFT

| FFT | Ось | Длина | Инструмент | Когда |
|-----|-----|-------|-----------|-------|
| дальностный | Z (быстрое время) | ~10000 | rocFFT (radix) | один раз, глобально |
| угловой | X, Y (апертура) | 16×16 | WMMA (тензор) | на каждом бине дальности |

Нарезка `окно 16, шаг 8, overlap 50%` — не нужна: после дечирпа цель это тон на всю запись, STFT дальность не локализует. Кубы `16×16×16` остаются только как тайлинг под GPU (16 бинов дальности в LDS), FFT по третьей оси куба не делаем.

**Важно про магнитуду:** до углового FFT данные держим **комплексными** — угловой FFT формирует луч когерентным сложением по апертуре, ему нужна фаза между 256 элементами. Магнитуда `|·|²` берётся только ПОСЛЕ углового FFT (в Кернеле B, см. §6).

---

## 3. Термины: девиация ≠ база

- Девиация `ΔF` (полоса ЛЧМ) → разрешение по дальности: `Δr = c/(2·ΔF)`.
- База `B = T_c·ΔF` (длительность × девиация), безразмерная → помехозащищённость, НЕ разрешение.

Формула `Δr = c/(2B)` неверна: в знаменателе `ΔF`, не база.

---

## 4. Метрика (предположительные числа)

Входы: `ΔF ≈ 6 МГц`, `f_s = 2 МГц`, `N = 10000`, UHF `0.5 ГГц` → `λ = 0.6 м`, `d = λ/2 = 0.3 м`.

```
T_c = N / f_s      = 10000 / 2·10⁶      = 5 мс      (длительность импульса)
Δr  = c / (2·ΔF)   = 3·10⁸ / (2·6·10⁶)  = 25 м      (разрешение по дальности)
B   = T_c · ΔF     = 5·10⁻³ · 6·10⁶     = 30000     (база)
```

Угол при `d = λ/2`:

```
sinθ = k / 8       (k = -8 … 7 после fftshift)
покрытие — полусфера, разрешение по sinθ = 1/8 (≈7° у визирования)
```

Вариант дальней связи: `ΔF = 500 МГц` → `Δr = 0.3 м`. Меняется только число.

---

## 5. Разрешение по дальности — держим все 10000

Не прореживаем:
- угловой FFT дёшев → экономить нечего;
- прореживание (8–16 бинов в один) размазывает энергию точечной цели → ≈ −10 дБ чувствительности.

`2^n` не обязателен: `10000 = 2⁴·5⁴`, rocFFT делает смешанным radix. Чистый radix-2 = добить до 16384.

---

## 6. Конвейер Шага 1 — два кернела

**Кернел A — дальностный FFT.**
```
rocFFT по Z, длина ~10000, батч 256 каналов
  load-callback на входе:  x[z] · w[z] (окно Хэнн/Блэкман) · ref_dechirp[z]
  выход:  КОМПЛЕКС  (магнитуду НЕ берём — впереди угловой FFT)
  (нормировку 1/N при желании на store-callback)
```
→ буфер `16×16×10000` комплексный, где Z = дальность.

**Кернел B — фьюзед WMMA (начинается в Шаге 2).**
```
один workgroup = 16 бинов дальности в LDS
  1. WMMA: угловой FFT 16×16 (комплекс)
  2. |·|²  → карта энергии 16×16  ← магнитуда здесь, переход в float
  3. линейка признаков (редукции в LDS)
  4. MLP-триаж → метка + скор
  5. на выход: токен (только выжившие)
```

Между кернелами — комплексный буфер в VRAM.

**Оценка времени** — считается позже, когда зафиксируем Кернел B (число признаков, размер MLP, стоимость окна). Старая прикидка «только два FFT» неполна: добавились callback-окно, `|·|²`, признаки, MLP. Замер на gfx1201.

## 7. Доплер (на будущее)

- В одном импульсе цель = один пик, несёт дальность и доплер смешанно; отделить нельзя.
- Доплер живёт по медленному времени (между импульсами). Пачка `P` импульсов, период `PRI`:

```
16 × 16 × N_дальность × P        (4-я ось = медленное время)
FFT по 4-й оси → доплер
Δf_d = PRF/P ,  Δv = λ·PRF/(2P) ,  однозначная v = λ·PRF/2
```

- Range walk: движущаяся цель уходит по дальности за пачку → компенсировать (keystone).
- По ТЗ: ЛЧМ Range-Doppler подсказывает доплер-банк M-коррелятора (~43 слепых бина → единицы).

Порядок осей: `угол → угол → дальность → доплер`. Первые три — Шаг 1, четвёртая — пачкой.
