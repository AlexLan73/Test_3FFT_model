# Алгоритм для инженеров — four-step FFT на тензорных ядрах

> Технический разбор к `fourstep_v2.py`, `fourstep32_compare.py`, `fourstep32_fp16_dr.py`.
> Аудитория: DSP/GPU-инженер. Формулы, сложность, отображение на железо, roofline.

---

## 1. Постановка

Приёмный тракт даёт комплексный тензор **A[x, y, z]** размера `Nap×Nap×Nz`
(апертура × апертура × быстрое время). Нужен куб **угол×угол×дальность**:

- **дальность** ← DFT по `z` (после дечирпа ЛЧМ частота биений `f_b = μ·2R/c` ∝ дальность);
- **угол×угол** ← 2D-DFT по апертуре `(x,y)` — пространственный beamforming при шаге `d=λ/2`:
  `sinθ = k·λ/(Nap·d) = 2k/Nap`, разрешение `Δsinθ = 2/Nap`.

Ограничение железа: матричные ядра (WMMA RDNA4 / MFMA CDNA) нативно перемножают блоки
**16×16** (RDNA4) или **16×16 и 32×32** (CDNA). Цель — реализовать угловой DFT **целиком**
через эти примитивы.

---

## 2. Four-step (Bailey, 1990) — вывод

DFT длины `N = N₁·N₂`. Индексация вход/выход:

```
n = N₂·n₁ + n₂ ,   n₁∈[0,N₁), n₂∈[0,N₂)
k = N₁·k₂ + k₁ ,   k₁∈[0,N₁), k₂∈[0,N₂)
```

Подставляем в `X[k] = Σₙ x[n]·W_N^{nk}`, `W_N = e^{-2πi/N}`:

```
X[N₁k₂+k₁] = Σ_{n₂} W_N^{n₂k₁} · W_{N₂}^{n₂k₂} · [ Σ_{n₁} x[N₂n₁+n₂] · W_{N₁}^{n₁k₁} ]
             └─twiddle─┘  └─stage2─┘              └────────── stage1 ──────────┘
```

Три множителя после раскрытия `W_N^{(N₂n₁+n₂)(N₁k₂+k₁)}`:
- `W_N^{N₂n₁·N₁k₂} = W_N^{N·n₁k₂} = 1` (исчезает);
- `W_N^{N₂n₁k₁} = W_{N₁}^{n₁k₁}` → **стадия 1** (DFT по n₁);
- `W_N^{n₂k₁}` → **twiddle** `TW[k₁,n₂] = e^{-2πi·k₁n₂/N}`;
- `W_N^{n₂·N₁k₂} = W_{N₂}^{n₂k₂}` → **стадия 2** (DFT по n₂).

**Псевдокод (ровно как в `fourstep_v2.py`):**

```python
xr = x.reshape(N1, N2)          # n = N2*n1 + n2
A  = fft(xr, axis=0)            # стадия 1 → [k1, n2]
B  = A * TW                     # TW[k1,n2] = exp(-2πi·k1·n2/N)
C  = fft(B, axis=1)             # стадия 2 → [k1, k2]
X  = C.T.reshape(N)             # k = N1*k2 + k1  (натуральный порядок)
```

2D — сепарабельно: four-step по строкам, затем по столбцам.
Проверка: отн. ошибка **8.9e-17** (fp64) — тождество, не аппроксимация.

## 3. Сложность и число проходов

Для 1D длины `N`, факторизация в `m` стадий (произведение = N):

| величина | формула | 256=[16,16] | 1024=[32,32] |
|----------|---------|-------------|--------------|
| стадий | `m` | 2 | 2 |
| twiddle-проходов | `m−1` | 1 | 1 |
| primitive-FFT работа | `Σ N/fᵢ` | 32×FFT-16 | 64×FFT-32 |
| twiddle-умножений | `(m−1)·N` | 256 | 1024 |

**Ключ для GPU:** каждый twiddle-проход на 2D = **чтение+запись всего куба** по VRAM.
Задача memory-bound → минимизация `m` важнее, чем FLOP. Отсюда выбор плитки (см. §6).

## 4. Отображение на матричные ядра

DFT-P = умножение на матрицу Фурье `F_P` (P×P, `F[a,b]=W_P^{ab}`). Батч из P векторов:

```
[P×P DFT] × [P×P данные] = один MFMA/WMMA-темп (P=16 нативно RDNA4; P=16/32 нативно CDNA)
```

Комплекс раскладывается на 4 вещественных matmul (`Fr,Fi × xr,xi`), аккумуляция в fp32.
Стадия 1 и стадия 2 four-step — обе суть батч-DFT-P → **обе на матричных ядрах**.
Twiddle — поэлементное комплексное умножение (vector ALU, между matmul-проходами).

Итог по 2D-256 (из `fourstep_v2.py`, §4): 16384 FFT-16 + 131072 twiddle-умн. на срез;
доля FFT-части на тензорных ядрах = **100%**.

---

## 5. Две стадии = развёртка неоднозначности (CRT по основанию N₁)

`k = k₁ + N₁·k₂` — это позиционная запись угла по основанию `N₁`:

- **стадия 1** (децимация ×N₂, эфф. шаг `N₂·d`): даёт `k₁` — тонкий отсчёт (`Δu=2/N`),
  но апертура прорежена → **grating lobes** с периодом `1/N₂` по `sinθ`. Неоднозначно ×N₁.
- **стадия 2** (сплошная плитка N₁): даёт `k₂` — грубый (`Δu=2/N₁`), **однозначный** отсчёт,
  который выбирает нужную реплику стадии 1.

Демо (`fourstep_v2.py` §3): источники A и B, разнесённые ровно на период grating lobe
(`Δu=1/8`), дают **одинаковый** `k₁=9`, но **разный** `k₂` (1 vs 2) → `k=k₁+16k₂` = 25 vs 9,
восстановление верное. Ни одну стадию убрать нельзя.

## 6. Плитка 16×16 vs 32×32 (`fourstep32_compare.py`)

| метрика | 16×16 | 32×32 | замер |
|---------|-------|-------|-------|
| точность four-step | 8.9e-17 | 6.7e-17 | = |
| стадий на N=1024 | 3 (16·16·4) | **2** (32·32) | −1 twiddle-проход |
| Δsinθ одной плитки | 1/8 (6.3°) | **1/16 (3.1°)** | ×2 острее |
| LDS/плитку (complex64) | 2 КБ | 8 КБ (12% из 64) | обе на чипе |
| нативность | RDNA4+CDNA | **только CDNA** | ⚠️ гейт по железу |

LDS-бюджет (complex float2 = 8 Б): 16²→2 КБ, 32²→8 КБ, 64²→32 КБ (½ LDS), 128²→VRAM.
Патент (гл.4): плитка держится на чипе вплоть до 64×64.

**Вывод:** для апертур-степеней-32 32-плитка = меньше проходов (memory-bound выигрыш) +
вдвое тоньше угол на нативном ядре. **Условие:** 32×32 нативна только на CDNA (MI100).
На RDNA4 (RX 9070) 32 эмулируется 4× блоками 16×16 — остаётся лишь выигрыш по стадиям.

## 7. Точность пониженной разрядности (`fourstep32_fp16_dr.py`)

DFT-32 = matmul с округлением входов к fp16/bf16 + **fp32-аккумуляция** (модель MFMA).
Замерено (ошибка против fp64):

| формат | мантисса | worst-case (широкополос.) | точечная цель | four-step-склейка |
|--------|----------|---------------------------|---------------|-------------------|
| fp32 | 23 бит | −142 дБ | −78 дБ* | 5.8e-8 |
| **fp16+fp32acc** | 10 бит | **−72 дБ** | −78 дБ* | 1.1e-4 (−79 дБ) |
| bf16+fp32acc | 7 бит | −55 дБ | −65 дБ | 1.5e-4 |

\* ограничено внесённым шумом сцены −60 дБ (fp16 не ухудшает).

Оценка «на бумаге» `√P·ε` (P=32): fp16 `√32·2⁻¹¹≈2.8e-3→−51 дБ`. Замер **−72 дБ** лучше —
**fp32-аккумуляция** гасит накопление (без неё было бы ~−51). **fp32-acc обязательна.**

**Решение:** угловой FFT в **fp16+fp32acc** (ДД −72 дБ ≫ требуемых −40…−60). Следствие —
можно хранить куб в **fp16** (4 Б/точка) → вдвое меньше трафика VRAM. bf16 — только детект.
Дальностный FFT (длинный, N~10³–10⁶) оставить **fp32** (в fp16 просядут дальностные боковики).

## 8. Конвейер ЛЧМ и roofline (MI100)

**Два раздельных FFT** (не 3D-FFT), патент гл.3:

```
A[16×16×N] ─[Kernel A: rocFFT по Z, fp32, load-cb: окно·дечирп]─▶ complex-куб (Z=дальность)
           ─[Kernel B: fused в LDS, per range-bin]────────────▶ токены
              1. WMMA/MFMA угловой DFT (four-step, fp16+fp32acc)
              2. |·|² → магнитуда (float, fftshift)
              3. инвариантные признаки (редукции в LDS)
              4. MLP-гейт → метка+скор
              5. write-out: только выжившие токены
```

**Roofline, куб 256×256×6000 = 393 Мточек, MI100 (BW 1.23 ТБ/с, fp16-MFMA 184.6 TFLOP):**

| стадия | лимит | оценка |
|--------|-------|--------|
| A. range FFT (fp32) | BW: вход fp16 1.57 + выход fp32 3.15 ГБ / 1.23 | ~4 мс |
| B. angular fp16 + magnitude (fused LDS) | BW: read 3.15 ГБ / 1.23; compute 200 GF / 184 TF ≈1 мс | ~3–4 мс |
| **итого** | **memory-bound** | **~7–11 мс/куб (~100 куб/с)** |

Арифметическая интенсивность угловой стадии: `200 GFLOP / 3.15 ГБ ≈ 63 FLOP/байт`.
Ridge point MI100 `≈ 184e12/1.23e12 ≈ 150 FLOP/байт` → **63 < 150 → memory-bound**.
Тензорные ядра недогружены в ~3–4×. Рычаги: **fused-LDS** (убрать промежуточный VRAM) и
**fp16-хранение** (−½ трафик), а не оптимизация FLOP.

Сравнение платформ (та же задача): RX 9070 ~20–40 мс; MI100 fp32 ~10–18 мс;
MI100 fp16 **~7–11 мс**.

## 9. Расширение на 3D-FFT 256×256×D (`fft3d_tensor.py`)

3D-DFT сепарабелен → три осевых DFT в любом порядке; осевой DFT = батч-умножение на `F_P`
= GEMM. **Все три оси на матричных ядрах.** Две категории осей:

| ось | размер | метод | обоснование |
|-----|--------|-------|-------------|
| x, y | 256 | four-step 16×16 / 32-tile | большой FFT из нативных тайлов (§2, §6) |
| **z** | **D=8..128** | **прямой DFT-GEMM** `F_D·X` | малая ось → 1 GEMM, без twiddle/reorder |

### Прямой DFT-GEMM vs four-step по оси z — точка перехода

`F_D·X` — стоимость `O(D²)` cMAC/эл, но **один проход** по памяти. four-step — `O(√D·2·D)`
cMAC, но **два прохода** + twiddle. Задача memory-bound → при малом D один проход побеждает.

| D | прямой (D²) | four-step (√D·2·D) | плитки | рекоменд. |
|---|-------------|--------------------|--------|-----------|
| 8 | 64 | 48 | 8×8 в 16×16 (25% K) | прямой |
| **16** | 256 | 128 | **1 нативная 16×16** | прямой |
| 32 | 1024 | 384 | 1 MFMA (CDNA) / 2×2 (RDNA) | прямой |
| 64 | 4096 | 1024 | 4×4 сетка 16×16 | прямой / four-step |
| 128 | 16384 | 3072 | 8×8 сетка 16×16 | four-step 16×8 |

Порог ~D=64. Доля z в общем cMAC: D=8→11%, D=32→33%, D=64→50%, D=128→67% (прямой).

### Замеры

- **Точность (vs `np.fft.fftn`)**: отн. ошибка 1.6e-15 (D=8) … 2.9e-14 (D=128) — машинный ноль.
- **Память**: 8.4 МБ (D=8) … 134 МБ (D=128) — ≪ дальностный куб 3 ГБ; батч в VRAM/L2.
- **fp16+fp32acc по z — ДД −72 дБ, D-НЕЗАВИСИМ** (−71.4…−72.0 дБ на D=8..128).
  Роста ~√D нет: fp32-аккумуляция гасит накопление по сумме, остаток = входное округление
  fp16 (per-element, D-инвариантно). → ось z в fp16 при любом D≤128 без fp32.

### Раскладка

```
256×256×D  ─ ось x (256): four-step 16×16/32-tile ─┐
           ─ ось y (256): four-step               ├─ GEMM, fp16+fp32acc, 100% MFMA
           ─ ось z (D):   прямой DFT-GEMM (D≤64)   ─┘
                          four-step 16×8 (D=128)
```

**Связь с патентом:** это НЕ 3D-FFT патента (там range-FFT и angular-FFT РАЗДЕЛЕНЫ, гл.3).
Честный 3D-FFT актуален для доплер-оси `16×16×N×P → FFT по 4-й` (гл.3.5, P=D — пачка
импульсов). При доплер-селекции с низкими боковиками ось z поднять в fp32 (ДД < −72 дБ).

## 10. Сводка проектных решений

1. Угловой FFT — **four-step из нативных плиток** (машинно-точен, 100% на матричных ядрах).
2. Плитка: **32×32 на MI100** (CDNA), **16×16 на RX 9070** (RDNA4). Параметр `(P1,P2)`.
3. Разрядность: угловой — **fp16+fp32acc** (−72 дБ); дальностный — **fp32**; куб — **fp16**.
4. Узкое место — **VRAM BW** → приоритет fused-LDS-кернел (патент гл.3.4), не арифметика.
5. Две стадии four-step несут развёртку grating-lobe неоднозначности «бесплатно».
6. **3D-FFT** = три GEMM: оси 256 через four-step, малая ось z (D≤64) — прямым DFT-GEMM
   одним matmul; fp16 по z держит −72 дБ независимо от D.

**Открытый пункт (для замера на живой MI100):** rocFFT-bench дальностной стадии +
профиль занятости MFMA в fused-кернеле — заменить roofline-прикидку на факт.
