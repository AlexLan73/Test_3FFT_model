# delegate-sonnet — протокол делегирования Sonnet (radar3d)

Реализацию (написание кода) отдаём **Sonnet** (беречь токены Opus), Кодо (Opus) пишет ТЗ,
делает **глубокое ревью** диффа и координирует. Триггер: `/delegate-sonnet` или «отдай Sonnet».

## 🔹 Схема (жёстко)

`Кодо разведка/ТЗ → Sonnet код (Agent, фон) → глубокое ревью Кодо (дифф + сам гонит тесты) → приём/коммит`

- **Разведка ДО ТЗ (Кодо):** прочитать реальные файлы/патент, найти существующее (НЕ плодить),
  сверить контракт. ТЗ пишется по факту кода, не по памяти.
- **Запуск:** `Agent(subagent_type: "general-purpose", model: "sonnet", run_in_background: true)`.
- **Ревью (Кодо, не на веру):** прочитать `git diff`, **самому** прогнать тесты через
  `.venv/Scripts/python.exe`, сделать независимый smoke ключевого поведения.
- **Обрыв связи:** агенты иногда падают на обрыве (API error). Кодо **дочитывает состояние**
  (`git status`/`git diff`) и **доделывает сама** — не перезапускает с нуля.
- **Не верить отчёту на слово** — цифры Sonnet перепроверять.

## 🔹 Шаблон ТЗ (секции — заполнять все)

```
# КОНТЕКСТ / ПРОБЛЕМА
<что и зачем, ссылка на патент-главу/спеку>

# ЖЁСТКИЕ ПРАВИЛА
🚫 pytest — только common.runner.TestRunner+AssertionGroup+SkipTest (правило 04).
🚫 НЕ писать в .claude/worktrees/ (правило 03) — только корень репо.
🚫 scipy/ruff/GPU/torch НЕТ в среде (дом) — опциональное под SkipTest, не чинить.
🚫 фронт/панель/сокет не трогать (если задача про базу).
Python≥3.11, type hints, docstrings по-русски, ООП/SOLID (Strategy/ABC/VO как в соседях).
Не мутировать входы. Детерминизм (без random/Date.now в логике). Имена строчные ASCII.
Интерпретатор: .venv/Scripts/python.exe (обычный python в PATH без numpy).

# СНАЧАЛА ПРОЧИТАЙ (не гадай про API)
<точные файлы: реализация-сосед, контракт-VO, образец-теста, патент-глава>
Проверь grep что сущность ещё НЕ существует (не плодить).

# ЧТО СДЕЛАТЬ
<нумерованные пункты: файлы, классы, контракты, точные формулы/сигнатуры>

# ПРОВЕРКА (обязательно, привести вывод)
.venv/Scripts/python.exe tests/<новый>.py
.venv/Scripts/python.exe tests/all_test.py 2>&1 | grep -E "[1-9][0-9]* fail" || echo "весь бэкенд 0 fail"
+ smoke ключевого поведения (в отчёт, не коммить).

# ОТЧЁТ
1. Что создано/изменено. 2. Вывод проверок (ok/fail/skip). 3. Smoke. 4. Честные девиации.
```

## 🔹 Приёмка (чек-лист Кодо после Sonnet)

- [ ] `git diff` прочитан, правки соответствуют ТЗ (без «отсебятины»).
- [ ] Затронутые наборы + `all_test.py` зелёные (прогнать самому).
- [ ] Независимый smoke ключевого поведения сделан (не только тесты Sonnet).
- [ ] Новый набор зарегистрирован в `tests/all_test.py`.
- [ ] Существующее не сломано (регресс), контрольные виды целы.
- [ ] Обновить `MemoryBank/tasks/IN_PROGRESS.md` (+ спека, если слой новый).

## 🔹 Масштаб — под задачу

Мелкая правка (девиация/докстринг) — короткое ТЗ. Новый слой (модуль+тесты) — полное ТЗ по шаблону.
Крупная фича — разбить на этапы, по агенту на этап, ревью каждого.

## 🚫 Чего НЕ делать
- ❌ Не запускать Sonnet без разведки (прочитать код/контракт ДО ТЗ).
- ❌ Не коммитить/пушить без явного «закоммить/запушь» от Alex.
- ❌ Не верить цифрам Sonnet — перепроверять самому.
- ❌ Не делать `git add -A` не глянув `git status` (чужие файлы из параллельного чата, см.
  память `two-chats-base-vs-demo`).
