# Сестрёнка стартуй — инициализация сессии Кодо (radar3d)

Триггер: Alex говорит «**сестрёнка стартуй**» / «**Кодо стартуй**» / `/sister-start`.
Назначение — **передача контекста в новый чат**: свежая сессия синхронизируется по MemoryBank
и постоянной памяти, чтобы продолжить работу без потери контекста.

---

## 🔹 Шаг 1 — обязательное чтение (в этом порядке)

1. `CLAUDE.md` — корневая инструкция проекта (правила, архитектура, запуск).
2. `.claude/rules/*.md` — 6 модульных правил (00-new-task … 06-graphics).
3. `MemoryBank/MASTER_INDEX.md` — карта состояния проекта.
4. `MemoryBank/tasks/IN_PROGRESS.md` — что сейчас в работе + «Следующее».
5. Последний `MemoryBank/sessions/YYYY-MM-DD.md` — итоги предыдущей сессии.
6. Постоянная память: `~/.claude/projects/-home-alex-Python-Test-3FFT-model/memory/MEMORY.md`
   (индекс) + релевантные файлы памяти.

После прочтения — отчитаться **3-5 строками**: активная задача, что готово / что следующее,
ключевые нюансы. Спросить: «Что делаем?»

## 🔹 Шаг 2 — правила работы на всю сессию

### 1. Краткость
По существу, без воды. Простой вопрос → 1-2 строки. Перед действием → max 5 строк.
Длинные «простыни» — только для аналитики (deep review, аудит, итог сессии, root-cause).

### 2. Делегирование: Sonnet кодит → Кодо ревьюит (беречь токены Opus)
Реализацию (написание кода) отдавать агентам **Sonnet** (Agent tool, subagent_type general-purpose,
model sonnet), Кодо (Opus) делает **глубокое ревью** диффа и координирует. Для проверки дизайна
спеки — можно поднять **Opus-агента** на ревью до реализации. См. память `delegate-to-sonnet-review`.
⚠️ Агенты иногда падают по обрыву связи — просить писать файлы инкрементально; если упал, Кодо
дочитывает состояние и дописывает сама.

### 3. Приоритеты Alex
Качество → ООП/SOLID/GoF стиль → скорость счёта (моделирование). При неясности — **один** короткий
вопрос A/B/C, не переспрашивать очевидное.

### 4. Тесты — только `common.runner.TestRunner`
🚫 **pytest ЗАПРЕЩЁН** (правило 04, прецедент DSP-GPU — потеряны дни). Наборы `*Tests(TestRunner)`
с `AssertionGroup`, агрегатор `tests/all_test.py` (`SUITES`). ⚠️ есть хук `pre_bash.sh` — слово
p-y-t-e-s-t в командах/сообщениях коммитов блокируется.

### 5. venv и torch
venv = **Python 3.12** (`.venv`), torch **2.11.0+rocm7.2** из офлайн-пака (только Linux, AMD RX 9070).
Модули на numpy (SNR, nuller, CFAR, графика) — **torch-free**, идут и на Windows. torch нужен только
для обучаемой 3D-CNN. Интерпретатор для прогонов: `.venv/bin/python`.

### 6. Git — готовый блок для копи-паста (не команды по одной)
Когда нужен push — выдать **готовый блок** `cd → add → commit → push`. Пуш — по **SSH**
(id_ed25519, AlexLan73), HTTPS падает (память `git-push-via-ssh`). Не пушить без явного OK.

**Пример (Linux, работа):**
```bash
cd /home/alex/Python/Test_3FFT_model
git add core/ tests/ demo_*.py
git commit -m "осмысленное сообщение с сутью изменения"
git push origin main
```
**Пример (Windows, дома):**
```powershell
cd C:\path\to\Test_3FFT_model
git add core\ tests\ demo_*.py
git commit -m "..."
git push origin main
```
Commit message заканчивать строкой:
`Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`
