# Aperture refactor — оркестратор i×j / 2ⁿ / zero-pad (radar3d)

Триггер: `/aperture-refactor` или «рефактор апертуры» / «апертура i×j».
Назначение — провести смену концепции апертуры с фиксированного квадрата **16×16** на
прямоугольник **i×j** (нечётные оси), каждая ось = **2ⁿ**, недобор элементов добивается нулями
(**zero-pad** до ближайшей 2ⁿ). Угловой FFT паддит апертуру до 2ⁿ, шкала `sinθ=k/(N_pad/2)` по каждой
оси независимо.

**Единственный источник истины по объёму правок:**
📄 `MemoryBank/specs/aperture_ixj_2n_refactor_2026-07-17.md` (аудит 4 разведчиков + этапы E1–E6).
Всегда читать его первым — там таблицы `file:line → правка`, физика паддинга, разграничение
«WMMA-железо vs логическая апертура», список из 17 патентных .md.

## 🔹 Физика (не ошибиться в E2)

- Порядок в угловом тракте: **окно (Хэмминг по РЕАЛЬНЫМ элементам) → zero-pad до 2ⁿ → fft2 → fftshift**.
  Паддить ПОСЛЕ окна (иначе окно накроет нули). Эталон — дальностная ось `range_fft.py` (уже так).
- Оси `kx/ky` строить на **padded** длине `N_pad`, иначе форма спектра не сойдётся и sinθ неверен.
- `sinθ_x = k_x/(N_pad_x/2)`, `sinθ_y = k_y/(N_pad_y/2)`; при i≠j — **разное разрешение по X и Y** (это норма).
- `ArrayConfig.padded_shape()` (`array_config.py:30`) — уже готовый источник 2ⁿ-размеров, вызвать его.
- **НЕ трогать** описание WMMA/MFMA 16×16 (железо) и дальностный zero-pad (`n_fft`, ортогонален).
