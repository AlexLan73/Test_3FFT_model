# 00 — Workflow при новой задаче

> Правило без `paths:` — **грузится всегда**. Короткое, чтобы помнить в каждой сессии.

## Последовательность (строго)

```
формулируй вопрос
        ↓
Context7 MCP — доки релевантных библиотек (numpy / scipy / torch / matplotlib)
        ↓
WebFetch / URL — свежие статьи/доки от разработчиков
        ↓
sequential-thinking MCP — если задача сложная (архитектура, математика 3D-БПФ, варианты)
        ↓
GitHub MCP — референсный код (при необходимости)
        ↓
ТОЛЬКО теперь писать код
```

## Правило «не кодить сразу»

- Не начинаю с Write/Edit, пока не собран контекст.
- Новая тема → Context7 (даже если думаю «знаю»).
- Сомнение в API → прочитать исходник + Context7, не гадать.

## Исключения (когда пропускаем шаги)

| Случай | Пропускаем |
|--------|-----------|
| Тривиальная правка опечатки | всё, сразу Edit |
| Alex явно говорит «правь так-то» | Context7/URL (его инструкция — истина) |
| Продолжение уже открытой темы | повторные Context7-запросы |

## Что обязательно ВСЕГДА

- Читать реальный код (`Read`/`Grep`) **до** утверждений про API/классы/пути.
- Не плодить сущности: перед созданием класса — искать существующий.
- Не опираться на память прошлой сессии без сверки.
